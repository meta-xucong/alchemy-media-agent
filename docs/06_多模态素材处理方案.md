# 06. 多模态素材处理方案

## 1. 目标

把用户上传的图片、短视频、PDF、PPT、DOC、表格等素材转换成 agent 可理解、可引用、可追踪的结构化上下文，同时避免把大文件、敏感数据和无用噪声直接塞进模型上下文。

## 2. Asset 生命周期

```text
uploaded -> scanning -> stored -> extracting -> analyzed -> ready
                                      |-> failed
```

每个 asset 产生：

- `asset_id`：全局引用 ID。
- 原始文件：对象存储路径。
- 派生物：缩略图、页面图、抽帧、音频、转写、文本、低分辨率预览。
- `material_brief`：供 agent 使用的摘要。
- 权限和保留策略：tenant、project、owner、retention、consent。

## 3. 图片处理

### 3.1 提取内容

- 文件元数据：格式、尺寸、大小、色彩模式。
- 视觉摘要：主体、场景、风格、构图、色彩、光线、文字、品牌元素。
- 参考用途：风格参考、角色参考、商品参考、构图参考、色卡参考、禁用元素。
- 哈希：perceptual hash 用于重复检测和版本关联。

### 3.2 给模型的输入

- 需要精确编辑时传原图或模型 file_id。
- 只做风格参考时传缩略图 + 摘要即可。
- 多图参考时限制数量，避免上下文过载。

## 4. PDF 处理

PDF 是最适合保留视觉布局的文档格式。处理方式：

- 提取文本。
- 渲染关键页面为图片。
- 检测封面、品牌页、色板页、样例页、图表页。
- 将页面摘要写入 `material_brief.pages[]`。

建议：品牌手册、Campaign brief、视觉规范优先要求 PDF。

## 5. PPT/DOC 处理

PPT/DOC 通常包含文本、图片、图表、排版。MVP 方案：

- 默认提取文本与元数据。
- 对需要视觉保真的文件，后端转换为 PDF，再进入 PDF 流程。
- 对复杂 PPT 可生成每页截图，交给视觉模型总结。

限制：直接文档输入通常无法可靠保留嵌入图片和图表；必须转换 PDF 或页面图。

## 6. 表格处理

适合批量生成：SKU、标题、卖点、颜色、价格、目标渠道。

处理策略：

- 识别 header、字段类型、样例值。
- 对前 1000 行生成摘要和批量计划。
- 大表拆分页，按 batch job 执行。
- 每行生成一个 prompt variable set。

## 7. 短视频处理

MVP 不直接依赖视频理解模型，采用通用预处理：

- ffmpeg 抽帧：按场景变化或固定间隔。
- 提取音频并 ASR。
- 生成镜头摘要：场景、人物、动作、运动、转场、色调、节奏。
- 提取首帧/尾帧/关键帧，给视频 provider 使用。

如果后续供应商支持视频理解，可在 `VideoUnderstandingProvider` 中直接传视频文件。

## 8. Material Brief Schema

```json
{
  "asset_id": "asset_123",
  "asset_type": "pdf",
  "summary": "品牌手册，主色为墨绿和米白，强调自然、有机、高端。",
  "visual_style": {
    "palette": ["#0B3D2E", "#F4EDE0"],
    "lighting": "soft natural light",
    "composition": "editorial minimal layout"
  },
  "text_constraints": ["品牌名必须写作 Kizuna Matcha"],
  "reference_usage": "style_and_brand",
  "pages": [
    {"page": 1, "role": "cover", "summary": "品牌主视觉"},
    {"page": 5, "role": "palette", "summary": "色彩规范"}
  ],
  "risks": []
}
```

## 9. 上下文压缩策略

不要把所有素材全文丢给 agent。优先级：

1. 用户最新指令。
2. 用户选中的输出/素材。
3. Material Brief 的关键约束。
4. 品牌强约束。
5. 历史 prompt 和 prompt patch。
6. 低优先级历史对话摘要。

## 10. 安全与隐私

- 上传前端限制文件大小和类型。
- 后端做病毒扫描和 MIME 检测。
- 公开 URL 禁止直接作为长期存储；要转存对象存储。
- 素材默认私有，签名 URL 短期有效。
- 敏感素材进入模型前做 PII 检测和权限确认。
