# 04. 系统架构设计

## 1. 分层架构

```text
[Web/Chat UI]
   |  SSE/WebSocket + REST
[API Gateway / FastAPI]
   |  auth, tenant, rate limit
[Orchestration Layer]
   |-- Claude Code Orchestrator 控制面
   |-- OpenAI Agents SDK Runtime 运行面
[Domain Services]
   |-- Asset Service
   |-- Prompt Service
   |-- Generation Service
   |-- Evaluation Service
   |-- Cost & Quota Service
   |-- Safety Service
[Provider Adapters]
   |-- LLM Providers: OpenAI / Anthropic / Gemini / OpenAI-compatible
   |-- Image Providers: OpenAI GPT Image 2 / Gemini / others
   |-- Video Providers: Seedance / Veo / others
[Infra]
   |-- PostgreSQL
   |-- Object Storage
   |-- Redis Queue
   |-- Vector DB optional
   |-- Observability
```

## 2. 关键组件

### 2.1 API Gateway / Backend

负责认证、项目/会话管理、上传、任务创建、状态查询、SSE 推送、结果下载。

### 2.2 Orchestration Layer

- `RuntimeManagerAgent`：高频请求主入口。
- `ClaudeCodeOrchestrator`：复杂任务、研发/运维、跨 agent 审核与控制面。
- `ToolBroker`：向 agent 暴露受控工具，不直接泄露底层 provider key。

### 2.3 Asset Service

管理素材上传、病毒扫描、格式识别、元数据、派生物、缩略图、转码、抽帧、OCR/ASR/文档解析。

### 2.4 Prompt Service

维护提示词模板、风格 preset、品牌约束、prompt patch、版本树、变量矩阵。

### 2.5 Generation Service

统一处理图片和视频生成任务。图片可以同步或异步；批量和视频必须异步。

### 2.6 Evaluation Service

对生成结果做自动评分：需求符合度、风格一致性、主体一致性、文字可读性、安全风险、尺寸/格式合规。

### 2.7 Provider Registry

统一注册模型供应商，支持运行时切换、能力探测、降级、熔断、重试、限流、成本估算。

## 3. 用户请求链路

1. 用户发送消息和文件。
2. API 创建 `session`、`message`、`asset` 记录。
3. Asset Service 解析文件，生成 `material_brief`。
4. Manager Agent 判断任务类型：生图、修改、批量、视频、咨询、澄清。
5. 若为生图，handoff 到 Prompt Architect 和 Image Generation Agent。
6. Image Generation Agent 调用 `generate_image` tool。
7. Generation Service 选择 provider 并创建任务。
8. 结果落对象存储，数据库记录 output。
9. Evaluation Agent 打分，UI 展示。
10. 用户继续炼金，Revision Agent 生成 prompt patch 并进入下一轮。

## 4. 数据流与控制流分离

- 数据流：大文件不进入 agent prompt；只传 file_id、asset_id、摘要、低分辨率预览或必要页面图。
- 控制流：agent 负责决策和工具调用；工具负责真实 API 调用和权限检查。
- 审计流：所有工具调用独立记录，不能只依赖模型 trace。

## 5. 批量任务架构

批量任务由 Batch Planner 拆成 `generation_jobs`：

```text
batch_request
  -> batch_plan
      -> job_001 prompt + params + asset_refs
      -> job_002 prompt + params + asset_refs
      -> job_003 prompt + params + asset_refs
```

队列 worker 并发执行，每个 job 有独立状态、错误、重试和输出。批次级别聚合进度和成本。

## 6. 视频任务架构

视频生成统一走异步任务：

```text
create video job -> submit provider task -> poll/webhook -> download/transcode -> ready
```

即使某个 provider 支持同步，也建议内部统一抽象为异步，避免 UI 和 API 被长请求拖死。
