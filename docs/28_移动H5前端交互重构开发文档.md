# 28. 移动 H5 前端交互重构开发文档

## 1. 背景与目标

当前移动 H5 的视觉方向已经成立：近白画廊底色、低饱和鼠尾草绿色、细线、轻边框、克制按钮和精致排版都符合手机版气质。

主要问题不在美术风格，而在信息架构：`高级版`、`案例展示区`、`2.0 历史`、`模型/API`、`组合提示词` 等区域仍然沿用桌面端的长页面和局部滚动容器。手机用户用手指操作时，很容易在主页面滚动和子区域滚动之间抢焦点，产生误滑、卡顿和定位困难。

本次重构目标：

1. 主页面只保留创作主线。
2. 复杂功能通过点击进入全屏子页、底部抽屉或详情弹层。
3. 保留当前 H5 已涵盖的全部功能，不因简化手机界面而删功能。
4. 避免主页面出现局部滚动条；长列表和长文本只在独立子页或详情页中滚动。
5. V1、V2、视频 demo 的功能入口更像手机 App，而不是桌面后台缩窄版。

核心设计语言：

```text
主页面负责创作。
子页面负责选择、配置、管理和诊断。
```

## 2. 设计原则

### 2.1 单一主滚动

主页面只允许浏览器页面本身上下滚动。不要在主页面中保留 `max-height + overflow:auto` 的复杂面板，例如历史网格、案例网格、组合提示词、Provider 列表。

允许滚动的场景：

- 全屏子页。
- 大图预览弹层。
- 运行详情页中的提示词文本框。
- 确实需要的底部抽屉内容，但抽屉高度和关闭动作必须清楚。

### 2.2 主流程短路径

用户打开手机页面后，最短路径应是：

```text
选择 V1 或 V2 -> 输入需求 / 上传素材 / 选模板 -> 生成 -> 查看结果
```

其他能力以入口卡片出现，不直接展开成大段控件。

### 2.3 复杂能力子页化

以下功能不应常驻主页面：

- 高级素材用途配置。
- 案例搜索、筛选和模板选择。
- V1 历史和 V2 历史完整相册。
- 模型/API 配置。
- V2 运行诊断、组合提示词、Claude 阶段详情。
- 事件日志。

### 2.4 拇指友好

所有可点击控件建议满足：

- 最小点击高度 44px。
- 删除、清空、覆盖等危险操作不放在缩略图卡片的主操作区。
- 主要动作固定在页面底部或子页底部，例如“使用此模板”“保存高级设置”“生成图片”。
- 列表卡片点击进入详情，详情页内再出现下载、删除、复制等操作。

### 2.5 状态摘要优先

主页面展示状态摘要，而不是完整配置项。

示例：

```text
高级素材：已上传 1 张 · 主体/Logo · 强参考
模板：已选「日式料理海报」· 框架锁定
参数：1 张 · 默认画幅 · PNG · 高质量
模型：GPT Image 2 · Claude Code 中枢
```

点击摘要卡进入完整设置。

## 4. 新信息架构

### 4.1 顶层结构

```text
H5 App
├─ V1 创作主页
│  ├─ 参数抽屉
│  ├─ 高级素材页
│  ├─ V1 历史页
│  ├─ V1 图片详情页
│  ├─ V1 设置页
│  └─ V1 事件页
├─ V2 创作主页
│  ├─ 案例选择页
│  ├─ V2 素材页
│  ├─ V2 参数/模板微调页
│  ├─ V2 运行详情页
│  ├─ V2 历史页
│  ├─ V2 图片详情页
│  └─ V2 设置页
└─ 视频 Demo 页
```

实现上不强制使用真实路由，可以在 `mobile.js` 中实现轻量 view stack：

```text
main view -> sub view -> detail view
```

并通过 `history.pushState` 支持手机返回键。

### 4.2 主页面结构

V1 主页面建议只保留：

1. 顶部品牌、模块切换、状态摘要。
2. 提示词输入。
3. 参数摘要卡：数量、画幅、格式、质量、强度。
4. 素材摘要卡：未上传/已上传/高级用途。
5. 生成按钮和生成状态。
6. 最近生成结果。
7. 最近作品入口：承接原首页生成历史轮播。
8. 灵感案例入口：承接原首页案例参考轮播。
9. 功能入口卡：历史、继续修改、设置、事件。

V2 主页面建议只保留：

1. 顶部品牌、模块切换、状态摘要。
2. 模板摘要卡：未选择/已选择/模板锁状态。
3. 提示词输入。
4. 素材摘要卡。
5. 参数摘要卡：数量、画幅、模板微调。
6. 生成按钮和 Claude 进度摘要。
7. 最近生成结果。
8. 最近作品入口：优先展示 V2 历史。
9. 灵感案例入口：进入 V2 案例选择页。
10. 功能入口卡：案例库、历史、运行详情、设置。

### 4.3 子页类型

| 子页类型 | 适用功能 | 行为 |
|---|---|---|
| 全屏子页 | 案例、历史、设置、素材、运行详情、事件 | 覆盖主页面，有顶部返回栏和底部主操作 |
| 底部抽屉 | 数量、画幅、格式、质量等短配置 | 从底部弹出，适合 1 屏内完成 |
| 详情页/详情弹层 | 图片预览、案例详情、历史详情 | 大图优先，操作放底部 |
| 确认弹层 | 删除、清空、覆盖 | 明确风险、二次确认 |

## 5. 页面级设计方案

### 5.1 V1 创作主页

主页面草图：

```text
Header
Mode Switch

Prompt Composer
[textarea]

Summary Cards
[参数 1张 · 默认 · PNG · 高质量]
[素材 未上传 / 已上传 1张 · 主体/Logo]

[生成图片]

Progress / Status

Latest Outputs
[最近 1-2 张图]

More
[最近作品] [历史图片] [继续修改] [灵感案例] [模型设置] [事件]
```

要求：

- `tuning-section` 不再作为主页面大段内容展示。
- `revision-section` 不再常驻大表单，改为结果详情或入口卡。
- `provider-panel` 和 `eventList` 不再常驻主页面。
- `historyGallery` 不在主页面内滚动，主页面只显示最近历史摘要。
- 原首页 `heroHistoryCarousel` 和静态 `case-showcase` 不能消失，应压缩成“最近作品”和“灵感案例”入口。

### 5.2 V1 参数抽屉

包含：

- 数量。
- 画幅。
- 输出格式。
- 图片质量。
- 工作强度。

交互：

- 点击主页面“参数摘要卡”打开。
- 底部固定“完成”按钮。
- 改动即时写入原有 state。
- 关闭后刷新摘要卡。

### 5.3 V1 高级素材页

包含：

- 素材上传。
- 素材预览。
- 高级用途。
- 参考强度。
- 保真级别。
- Logo 角标位置。
- 用途说明。
- 清空素材。

交互：

- 点击主页面“素材摘要卡”进入。
- 顶部显示当前素材状态。
- 底部固定“保存设置”。
- 如果未上传素材，只显示上传区和简短说明，不展开所有高级用途。
- 选择 Logo 后才显示角标位置。

### 5.4 V1 历史页

包含：

- 顶部返回、刷新、数量。
- 历史图片相册。
- 点击图片进入历史详情。
- 历史详情包含：大图、原始提示词、模型信息、下载、用于继续修改、删除。

要求：

- 历史页可以滚动，主页面不滚动历史容器。
- 删除不直接出现在网格卡片上，避免误触。
- 历史加载更多改为页内自然加载或底部“加载更多”。

### 5.5 V1 设置页

包含：

- 生图模型 GPT/Gemini。
- 思考模型 GPT/Kimi。
- API Key。
- Base URL。
- 模型 ID。
- Provider 列表。

要求：

- 主页面只显示当前模型摘要。
- API Key 输入仍然保留自动保存逻辑。
- 高级 API 配置可以折叠，但折叠发生在设置页内部，不在主页面里。

### 5.6 V1 事件页

包含：

- 事件数量。
- 事件列表。
- 最近 Trace。

要求：

- 不再占用主页面空间。
- 支持清晰的空状态。

### 5.7 V2 创作主页

主页面草图：

```text
Header
Mode Switch

Template Card
[未选择模板 / 已选模板 · 框架锁定] [选择]

Prompt Composer
[textarea]

[未上传素材 / 已上传 1张 · 主体/Logo]

Parameter Card
[1张 · 默认画幅 · 模板微调]

[生成图片]

Claude Progress Summary

Latest Output

More
[最近作品] [案例库] [历史] [运行详情] [设置]
```

要求：

- 保留 V2 Template Lock 原则：手选案例是最高优先级视觉锚点。
- 主页面只显示选中模板和锁定摘要，不直接铺开案例库。
- 主页面不直接展示 `v2PromptPlan` 长文本。
- 如果有 V2 最近生成图，主页面展示最多 1-2 张；完整历史进入 V2 历史页。

### 5.8 V2 案例选择页

包含：

- 顶部返回。
- sticky 搜索框。
- 匹配按钮。
- 大模型思考中状态。
- 标签筛选。
- 案例图库。
- 案例详情。
- 底部固定“使用此模板”。

交互：

- 点击主页面模板卡或“案例库”入口进入。
- 搜索和筛选只影响案例页。
- 点击案例卡先预览，不直接改变模板，避免误触。
- 只有点击“使用此模板”才写入 `v2State.selectedTemplateId`。
- 已选模板在案例页顶部显示，并提供“清空模板”。

### 5.9 V2 素材页

包含：

- 上传参考图片。
- 素材预览。
- 素材用途。
- 约束强度。
- 用途说明。
- 素材列表。
- 清空素材。
- 视觉语法锁提示。

要求：

- 如果已选模板，页面顶部明确显示“模板控制框架，素材填入可替换位置”。
- 上传素材后自动刷新主页面素材摘要。
- 保留 hard identity 资产需要作为 provider input images 的提示，不在前端弱化为普通文案。

### 5.10 V2 参数/模板微调页

包含：

- 生成数量。
- 画幅。
- 替换主体。
- 想要的感觉。
- 使用场景。
- 清空模板选择。

要求：

- 默认画幅仍保持空值，让引擎默认生成。

### 5.11 V2 运行详情页

## 9. 风险与注意事项

1. 不要复制带 ID 的 DOM 节点，避免事件绑定混乱。
2. 移动 DOM 后要确认原有 `els.*` 引用仍有效。
3. 历史和案例列表从主页面迁移后，渲染函数不应假定父容器在主页面。
4. `setupH5AdvancedPanels()` 必须改造，否则会继续把复杂区块塞回主页面。
5. 浏览器返回键要关闭子页，不能直接离开网页。
6. 删除按钮需要 `event.stopPropagation()`，避免触发图片预览。
7. V2 运行中进入详情页后，轮询刷新仍要更新详情页内容。
8. 设置页输入 API Key 时，要避免因为子页关闭而丢失 autosave。
9. 主页面摘要要从真实 state 派生，不要维护第二套独立状态。
10. 不要为了视觉简洁隐藏错误信息，错误要在当前操作层级可见。

## 10. 自审第一遍：功能完整性查漏

对照当前 H5，已覆盖：

- V1 主生图链路。
- V1 基础参数。
- V1 高级素材。
- V1 结果、继续修改、历史、事件、模型/API。
- V2 案例库、模板选择、Template Lock。
- V2 Agent 生图、素材、参数、模板微调。
- V2 运行详情、组合提示词、Claude 进度。
- V2 历史、Provider、模型、Kernel、高级调度。
- 视频 demo。
- 示例弹层、大图预览、提示词复制、下载、删除。

补充要求：

1. V2 主页面必须保留“当前模板”和“当前素材”的摘要，否则用户离开子页后会失去上下文。
2. 历史详情要支持“用于继续修改”，这是 V1 炼金闭环的一部分。
3. V2 运行详情要在生成中也可打开，不只生成后可看。
4. 设置页要保留运行时 autosave，不要求用户额外保存 API Key。
5. 首页原有的历史轮播和案例参考不能删除，只能手机化为入口卡或轻量摘要。
6. 空状态也属于功能：历史为空、案例为空、素材为空、输出为空时要有明确提示和下一步动作。

## 11. 自审第二遍：手机交互查漏

容易遗漏的手机体验问题：

1. 案例页搜索框应该 sticky，否则用户滚到底部后难以重新搜索。
2. 子页底部按钮要适配 iPhone safe area。
3. 历史页和案例页的卡片操作不要太密，卡片主体只负责预览。
4. 组合提示词长文本必须放在详情页固定框中滚动，不能撑爆页面。
5. 主页面入口卡要显示配置摘要，不能只写“高级”两个字。
6. 生成中状态要在主页面和运行详情页同步，避免用户误以为卡住。
7. 返回子页后焦点应回到打开入口，方便连续操作和可访问性。
8. 模块切换时应关闭当前子页或确认是否丢失当前子页上下文。
9. 弹层、子页、底部抽屉不能同时打开多层；打开新层前要明确关闭或压栈。
10. 所有子页都要给出空状态下一步，例如“去选择案例”“上传素材”“刷新历史”。

最终建议：

```text
先重构信息架构，再打磨视觉。
先保证所有功能可达，再减少主页面噪音。
主页面越短，子页面越清晰，手机版才会真正成立。
```
