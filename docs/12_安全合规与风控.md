# 12. 安全合规与风控

## 1. 风险类别

| 类别 | 示例 | 防护 |
|---|---|---|
| 内容安全 | 色情、暴力、仇恨、违法 | 输入/输出审核、拒绝与改写建议 |
| 版权/IP | 明确要求生成受保护角色、商标滥用 | 风险提示、阻断、替代表达 |
| 肖像/隐私 | 上传真人照片要求敏感改造 | 权利声明、敏感场景限制 |
| 未成年人 | 儿童图像、危险场景 | 更严格策略 |
| 商业误导 | 虚假广告、医疗金融承诺 | 文案审核和免责声明 |
| 数据安全 | 上传合同、内部资料 | 租户隔离、加密、最小化上下文 |
| 密钥泄露 | API Key 硬编码、日志泄露 | Secret Manager、日志脱敏、hook 拦截 |

## 2. 用户素材权利声明

上传素材时建议加入 checkbox：

> 我确认我拥有或已获得上传素材的合法使用权，并授权系统在当前项目中用于内容理解、生成、编辑和存储。

对公共人物、商标、影视角色、品牌资产要额外提示。

## 3. 输入审核

在生成前检查：

- 明确违法、色情、仇恨、暴力请求。
- 侵犯个人隐私或身份冒充。
- 明确复刻受保护 IP 的请求。
- 高风险行业虚假声明。
- 用户上传素材是否含敏感个人信息。

## 4. 输出审核

输出后检查：

- 是否出现用户未请求的敏感内容。
- 是否出现可识别人物、品牌、文字错误。
- 是否违反平台规则。
- 是否需要 watermark/provenance 保留。

## 5. Provenance

OpenAI 生成图片包含 C2PA metadata 与 SynthID watermark。系统转存、压缩、导出时要尽量保留 metadata，并记录生成来源。如果导出流程会丢失 metadata，要在审计记录里说明。

## 6. 密钥安全

- 后端代调用，不让前端拿供应商 API Key。
- BYOK 使用 envelope encryption。
- key 只在调用时解密到内存，不写日志。
- Claude Code hooks 阻断包含密钥的文件写入或 shell 输出。
- provider key 按项目/环境隔离。

## 7. 文件安全

- 上传大小限制。
- MIME sniffing。
- 病毒扫描。
- 文件名归一化，避免路径穿越。
- 对外下载使用短期签名 URL。
- 默认私有桶。

## 8. 审计日志

必须记录：

- 谁在什么时候上传了什么类型素材。
- 谁发起了生成，使用哪个模型和 provider。
- 哪些请求被安全拦截。
- 哪个 API Key/项目被调用，成本估算多少。
- 管理员修改 provider 或安全策略的记录。

## 9. 人类审批

以下场景建议强制人工确认：

- 高额批量生成。
- 视频生成。
- 公开人物或强品牌商用素材。
- 用户上传包含敏感文档。
- 输出将自动发布到外部渠道。

## 10. 不绕过供应商安全

如果某 provider 因安全策略拒绝请求，系统不得自动切到另一个 provider 绕过限制。ProviderSelector 只能在技术错误、限流、不可用时降级。
