# V1/V2 极简模式前端开发文档

## 目标

在 V1 和 V2 现有专业工作台之外，新增一个隔离的“极简模式”入口。用户只需要输入一句需求，或上传参考图片，即可一键调用现有生成链路出图。

## 设计边界

1. 不改后台 API、数据模型、存储、鉴权、V1/V2 生成服务。
2. 专业模式默认保持现状，原有 V1/V2 前端布局、参数、素材、历史、星标、继续修改逻辑不搬家、不重构。
3. 新 UI 只放在极简模式面板里，通过“极简模式 / 专业模式”切换入口显示。
4. 极简模式运行时只临时使用前端默认值，不写入专业模式的素材状态、模板选择、星标参考图选择或高级参数。
5. V2 严格保持后端隔离：极简模式调用 V2 原生 `/api/v2` 上传与生成链路，不调用 V1 接口。

## 第一版功能

### 公共交互

- 每个 V1/V2 页签顶部新增一个模式切换条。
- 默认进入“专业模式”，避免影响当前用户习惯。
- 切到“极简模式”后，只显示极简输入面板、生成结果和历史等被动结果区域。
- 极简面板包含：
  - 需求文本框；
  - 图片上传框；
  - 已选图片数量与文件名；
  - 一键生成按钮；
  - 清空按钮；
  - 简短状态提示。

### V1 极简模式

- 使用现有 V1 上传能力上传参考图。
- 使用现有 V1 `/v1/image/jobs` 生成能力提交任务。
- 默认参数：
  - 数量：1；
  - 质量：high；
  - 格式：png；
  - 工作强度：balanced；
  - 画幅：默认；
  - provider：沿用当前 V1 生图 provider。
- 如果用户只上传图、不写需求，前端补一个保守默认需求：基于参考图生成一张高质量视觉图，保持核心主体和风格一致。

### V2 极简模式

- 使用 V2 原生 `/api/v2/uploads` 上传素材。
- 使用 V2 原生 `/api/v2/creative/runs/async` 提交任务。
- 默认不继承专业模式里的手选模板、星标参考图、模板微调字段。
- 默认让 V2 中枢自由检索案例并融合上传图片。
- 默认参数：
  - 数量：1；
  - 质量：high；
  - 格式：png；
  - provider：沿用 V2 当前有效生图 provider；
  - 提示词增强：交给 V2 后端 auto 逻辑。
- 如果用户只上传图、不写需求，前端补一个保守默认需求：基于参考图片生成一张高质量视觉图，保持核心主体和风格一致。

## 自动素材意图第一版

极简模式不暴露素材用途复选框，但前端会根据用户一句话做轻量规则判断：

- 包含 Logo、标志、商标、品牌字样：按 Logo 参考。
- 包含 人脸、头像、肖像、人物：按身份/主体参考。
- 包含 产品、瓶、包装、主体：按主体参考。
- 包含 背景、场景、环境：按背景参考。
- 包含 构图、版式、海报、布局：按构图参考。
- 无明显意图：按风格参考。

该判断只用于第一版前端自动填槽，不改变专业模式中用户手动选择的素材用途。

## 风险控制

1. 专业模式默认显示；极简模式需要用户主动切换。
2. 极简模式不复用 `state.assets`、`v2State.uploadedAssets` 作为持久 UI 状态，避免污染专业模式素材面板。
3. V2 极简模式提交时固定 `template_case_id: null`，防止隐藏模板误参与。
4. 生成结果仍走现有 `renderGallery` / `renderV2Run` / 历史刷新函数，避免新增第二套结果渲染。
5. 移动端同步添加独立极简主卡片，不依赖 H5 专业入口卡片搬运逻辑。

## 测试清单

1. 桌面 V1 默认仍显示专业模式。
2. 桌面 V1 切极简模式后，仅看到极简输入、结果与历史等结果区域。
3. 桌面 V1 一句话生成能提交任务。
4. 桌面 V1 上传图片加一句话能上传素材并提交任务。
5. 桌面 V2 默认仍显示专业模式。
6. 桌面 V2 切极简模式后，不显示案例、参数、素材专业设置。
7. 桌面 V2 极简模式生成时不继承专业模板和星标参考图。
8. 移动 V1/V2 均有模式切换与极简输入面板。
9. 极简模式清空按钮只清空极简输入，不清空专业模式已传素材或模板。
10. 浏览器控制台无前端语法错误。
