# 02. 实施蓝图 / 里程碑

## 总体原则

先做可用闭环，再做多模型，再做视频。不要一开始把所有 agent 框架都接进主链路，否则容易变成“框架动物园”。主路径要短：用户需求 → 素材理解 → prompt 规划 → 生图 → 评价 → 调整。

## 阶段 0：技术预研与项目脚手架

目标：确认供应商、权限、SDK、文件格式和基础架构。

交付物：

- Python/FastAPI 服务骨架。
- OpenAI Agents SDK 最小 agent loop。
- Claude Code/Claude Agent SDK 调度样例。
- OpenAI GPT Image 2 adapter PoC。
- 文件上传与对象存储 PoC。
- 基础数据库 schema。
- `.env.example` 和密钥管理方案。

验收：能用一条 prompt 生成一张图，并在数据库记录会话、任务、模型调用和输出文件。

## 阶段 1：生图 MVP

目标：完成类 ChatGPT 的单人单项目生图体验。

交付物：

- Chat UI：消息流、上传组件、生成进度、画廊展示。
- 生图 Agent：需求识别、prompt 优化、参数选择、GPT Image 2 调用。
- 素材解析：图片、PDF、PPT/DOC 文本、表格前 1000 行摘要。
- 任务队列：图片生成任务异步化，支持取消和重试。
- 炼金：选中图片后继续修改，形成版本树。
- Trace：请求、agent、模型、存储、错误串联。

验收：用户可上传参考图/PDF，生成 1-10 张图，选择某张继续修改并下载结果。

## 阶段 2：批量与多模型切换

目标：把系统从“能用”升级为“可运营”。

交付物：

- Batch Planner：变量矩阵、SKU 表、批量 prompt。
- Model Registry：OpenAI/Gemini/其他图片 provider 配置。
- Cost Guard：每个项目、用户、批次的预算和限额。
- 自动评价：Prompt adherence、文字可读性、主体一致性、素材引用一致性。
- 结果筛选：按评分、参数、模型、版本过滤。

验收：同一批需求可切换 OpenAI/Gemini 生成，批量任务可中断、重试、导出清单。

## 阶段 3：视频接口预留到可跑通

目标：视频先做到“任务接口可用”，产品功能可灰度。

交付物：

- `VideoProvider` 抽象。
- Seedance/Veo 至少一个 provider stub 或 PoC。
- 文生视频、图生视频、参考图生成视频任务创建。
- 任务查询、取消、下载、过期处理。
- 视频安全、版权、成本提示。

验收：通过 API 创建一个视频任务，轮询状态，拿到可下载 URL 或失败原因。UI 可显示“实验性视频生成”。

## 阶段 4：团队协作与生产化

目标：进入真实业务运营。

交付物：

- 组织/项目/成员权限。
- 品牌资产库与项目记忆。
- 提示词模板库、风格 preset、工作流模板。
- 人工审批流、评论、导出包。
- 灰度发布、A/B 测试、模型效果看板。
- 安全审计和合规报表。

验收：团队可共享项目素材和生成结果，管理员可查看费用、风险、模型表现。

## 建议排期

| 周期 | 重点 | 产出 |
|---|---|---|
| 第 1 周 | 预研与脚手架 | SDK PoC、数据库、对象存储、OpenAI 图片 adapter |
| 第 2-3 周 | MVP 主链路 | Chat UI、上传、单次生图、任务队列 |
| 第 4 周 | 炼金闭环 | 版本树、编辑、自动评价、下载 |
| 第 5 周 | 批量与多模型 | 批量任务、Gemini adapter、成本统计 |
| 第 6 周 | 视频预留 | VideoProvider、Seedance/Veo stub、API 文档 |
| 第 7-8 周 | 稳定化 | 安全、压测、回归、灰度、文档补齐 |

## 关键决策点

1. Claude Code 是否进入线上高频主路径。
   - 推荐默认：复杂任务和研发/运维中枢走 Claude Code；高频生图请求直接由 OpenAI Agents SDK manager agent 执行。
   - 可选严格方案：所有请求先进入 Claude Code Orchestrator，再调用 OpenAI Agents Runtime。

2. 素材解析是否用 OpenAI Responses file inputs。
   - MVP 可以直接用。
   - 对 PPT/DOC 中的图片/图表，要转换为 PDF 或页面图片来保留视觉信息。

3. 视频是否早期开放 UI。
   - 建议只给内部灰度用户开放，避免任务时长、成本、版权、安全风险过早暴露。
