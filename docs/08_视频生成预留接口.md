# 08. 视频生成预留接口

## 1. 目标

视频模块 MVP 只留接口和任务框架，不阻塞生图。设计目标是后续能接 Seedance、Veo 或其他视频生成模型，而无需大改 UI 和数据库。

## 2. 视频能力范围

预留以下任务类型：

- `text_to_video`：文生视频。
- `image_to_video`：单图/多图生视频。
- `reference_to_video`：参考图、短视频、PDF/PPT 摘要辅助生成。
- `extend_video`：视频续写。
- `first_last_frame_video`：首尾帧控制。
- `video_edit`：后续可选。

## 3. 为什么视频必须异步

视频生成耗时长、费用高、失败原因复杂、结果 URL 可能短期有效。统一做任务式 API：

```text
POST create -> provider task id -> polling/webhook -> download -> transcode -> ready
```

## 4. VideoProvider 接口

```python
class VideoProvider(Protocol):
    name: str
    async def capabilities(self) -> VideoCapabilities: ...
    async def create_task(self, request: VideoGenerationRequest) -> VideoTask: ...
    async def get_task(self, provider_task_id: str) -> VideoTaskStatus: ...
    async def cancel_task(self, provider_task_id: str) -> None: ...
    async def download_result(self, provider_task_id: str) -> bytes: ...
```

## 5. VideoJobRequest 示例

```json
{
  "task_type": "image_to_video",
  "prompt": "让这张咖啡海报变成 6 秒镜头，阳光移动，蒸汽缓慢升起，镜头轻微推进。",
  "asset_ids": ["asset_cover_image"],
  "duration_seconds": 6,
  "aspect_ratio": "9:16",
  "resolution": "1080p",
  "motion_style": "slow cinematic push-in",
  "audio": {"enabled": false},
  "provider_preference": "seedance"
}
```

## 6. VideoJob 状态机

```text
created -> queued -> submitted -> processing -> downloading -> transcoding -> ready
                 |          |            |             |             |
                 |          |            |             |             -> failed
                 |          |            |             -> expired
                 |          |            -> provider_failed
                 |          -> submit_failed
                 -> canceled
```

## 7. Provider 适配重点

### Seedance

- 任务创建、查询、列表、取消/删除。
- API Key 需要环境变量和地域隔离。
- 适合文本/图像/多模态参考生成视频的实验路径。

### Veo

- 支持文生视频、图像参考、首尾帧、不同分辨率和画幅。
- 可作为国际化部署或 Gemini 生态备选。

## 8. UI 预留

- 生图结果卡片增加“生成视频”按钮。
- 视频任务进入单独 tab：排队中、生成中、可下载、失败。
- 视频成本高，必须二次确认。
- 支持选择时长、画幅、运动风格、是否音频。

## 9. MVP 的返回策略

在视频未正式接入前：

- API 可创建 `experimental` 任务。
- 如果未配置 provider，返回 `provider_not_configured`。
- UI 展示“视频生成能力正在灰度，可先保存视频 prompt”。
- 任务不会吞掉用户需求：保留 prompt plan 和素材引用，后续可一键重跑。
