# 07. 生图工作流与炼金闭环

## 1. 首次生图流程

```text
User Prompt + Assets
  -> Intent Parse
  -> Asset Analysis
  -> Image Prompt Plan
  -> Provider Selection
  -> Safety Check
  -> Image Generation
  -> Postprocess
  -> Critic Score
  -> Gallery Display
```

## 2. Image Prompt Plan

生成前必须先形成结构化计划，而不是把用户原话直接丢给模型。

字段建议：

```json
{
  "main_subject": "一杯草莓抹茶拿铁",
  "scene": "布鲁克林街角咖啡店外带窗口",
  "style": "高端生活方式摄影，日系清爽，软自然光",
  "composition": "竖版小红书封面，主体居中，顶部留标题区",
  "brand_constraints": ["使用墨绿色和米白色", "避免廉价塑料质感"],
  "text": {"required": true, "content": "夏日限定", "language": "zh-CN"},
  "negative_constraints": ["不要多余手指", "不要错误品牌名"],
  "outputs": {"count": 6, "size": "1024x1536", "format": "png", "quality": "high"}
}
```

## 3. 图片生成参数

MVP 暴露给用户的参数要少，内部参数要完整记录。

用户可见：

- 数量：1、2、4、6、8、10。
- 画幅：方图、横图、竖图、自定义。
- 质量：快速、标准、高质量。
- 风格：摄影、插画、漫画、海报、电商、信息图等。
- 是否透明背景。

内部记录：

- provider、model、model version。
- prompt、negative prompt 或约束字段。
- size、quality、format、compression、background。
- input assets、reference roles。
- request id、trace id、cost estimate。

## 4. 批量生成

批量生成分两类：

### 4.1 同 prompt 多变体

适合探索风格。只改随机性、细节变量、构图变量。

### 4.2 变量矩阵

适合电商/广告。变量来自表格或用户指定：

```text
product x season x background x headline
```

Batch Planner 产出：

- `batch_id`
- `jobs[]`
- 每个 job 的 prompt plan。
- 并发策略。
- 成本上限。
- 失败重试策略。

## 5. 炼金 / 继续修改

炼金不是简单把用户反馈追加到原 prompt。推荐做 `PromptPatch`：

```json
{
  "base_output_id": "out_123",
  "user_feedback": "保持构图，把背景改成雪天，杯身品牌字更清晰",
  "preserve": ["main_subject", "composition", "brand_palette"],
  "change": ["background_weather", "logo_legibility"],
  "edit_mode": "image_edit",
  "new_prompt_delta": "Change the background to a calm snowy winter street while preserving the cup position, camera angle and brand palette. Make the cup logo crisp and readable."
}
```

## 6. 版本树

每次生成或编辑都形成版本节点：

```text
v1 initial generation
  |-- v1.1 edit background
  |-- v1.2 improve text
       |-- v1.2.1 change color
```

每个节点保存：

- 输入 prompt plan。
- 输出文件。
- 用户反馈。
- 修改 patch。
- 模型和参数。
- 自动评分和人工收藏。

## 7. 自动评价

Critic Agent 输出评分，但不替代用户判断。

评分维度：

| 维度 | 说明 |
|---|---|
| Prompt adherence | 是否符合用户需求 |
| Asset consistency | 是否遵守参考素材/品牌约束 |
| Text quality | 图中文字是否准确可读 |
| Composition | 构图是否适合目标渠道 |
| Subject integrity | 主体是否变形、缺失、伪影 |
| Safety | 内容是否安全、版权/肖像风险 |
| Technical | 尺寸、格式、透明背景等是否符合要求 |

## 8. UI 交互建议

- Chat 左侧/中间，结果画廊右侧或下方。
- 每张图展示：收藏、下载、继续修改、复用为参考、查看 prompt、删除。
- 批量结果支持：网格、筛选、批量下载、导出 CSV。
- 版本树用“分支”或“历史版本”面板展示。
- 对低分结果标记“可能有文字错误/风格偏差”，给出一键修复按钮。

## 9. 失败处理

| 场景 | 处理 |
|---|---|
| 模型超时 | 自动重试，超过阈值进入 failed，可手动重试 |
| 内容安全拦截 | 告知原因并建议安全替代表达 |
| 图片文字错误 | 建议继续修改或使用更适合文字的模型/参数 |
| 批量部分失败 | 批次仍完成，失败项可重跑 |
| 素材太大 | 后台解析，先返回“正在处理素材” |
