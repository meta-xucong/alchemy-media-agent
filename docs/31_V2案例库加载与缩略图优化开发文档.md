# 31. V2 案例库加载与缩略图优化开发文档

## 1. 背景

V2 案例库已经接入 GitHub 上游，并在生产环境中保持 7 天一次同步。当前前端打开 V2 时会请求 `/api/v2/templates?limit=1000`，虽然界面只先渲染一部分卡片，但浏览器仍然一次性获取大量案例元数据，并在用户继续浏览时逐步触发大量缩略图请求。

同时，案例缩略图当前由后端按 `360 x 450`、WebP quality `76` 生成。这个规格适合低带宽列表，但在桌面高分屏、大卡片、手机 DPR 2-3 场景下容易显得模糊，不利于用户判断案例构图、文字、产品细节和视觉风格。

本章优化只处理 V2 案例浏览体验：

- 不引入 CDN。
- 不改 V1。
- 不改 Sub2API。
- 不改 Veyra 账户、登录、扣费链路。
- 不改 V2 生图中枢、Template Lock、上传素材融合、历史记录或计费逻辑。

## 2. 目标

1. V2 初始化不再一次性拉取 1000 条案例。
2. 案例浏览改成“轻量索引 + 分页加载”。
3. 桌面端首批加载 24 个案例，移动端首批加载 16 个案例。
4. 分类筛选通过后端分页接口重新加载，避免前端必须持有全量案例。
5. 案例缩略图至少能看清主体、构图、文字和关键产品细节。
6. 旧 `/api/v2/templates` 和旧 `/api/v2/case-thumbnails/{path}` 保留，避免影响现有测试、内部调用和历史数据。

## 3. 非目标

- 不做 CDN 域名、对象存储或边缘缓存配置。
- 不把用户生成图片公开缓存。
- 不重写 `/api/v2/prompt-cases/search` 的搜索逻辑。
- 不改变 Claude Code 中枢如何选择模板或组合提示词。
- 不清理生产旧缩略图缓存。新旧缓存可并存，稳定后再人工清理。

## 4. 后端接口设计

### 4.1 轻量索引

新增：

```text
GET /api/v2/templates/index
```

返回：

```json
{
  "total": 804,
  "index_version": "github_evolinkai_gpt_image_cases:github-xxxx",
  "facets": [
    { "value": "poster", "count": 120 },
    { "value": "commercial", "count": 96 }
  ],
  "categories": [
    { "value": "poster", "count": 120 }
  ],
  "style_tags": [
    { "value": "cinematic", "count": 88 }
  ],
  "use_case_tags": [
    { "value": "product", "count": 76 }
  ]
}
```

说明：

- 只返回数量和标签统计，不返回长 prompt、视觉特征或图片内容。
- `facets` 是前端标签栏使用的合并统计，包含 category、style_tags、use_case_tags。
- 用于首屏展示总数、筛选项和当前案例索引版本。

### 4.2 分页案例

新增：

```text
GET /api/v2/templates/page?limit=24&cursor=0&facet=poster
```

参数：

| 参数 | 说明 |
| --- | --- |
| `limit` | 每页案例数，1-96。桌面默认 24，移动默认 16。 |
| `cursor` | 偏移游标。为空时从 0 开始。 |
| `facet` | 可选，分类或标签筛选。`all` 等同不筛选。 |
| `category` | 可选，兼容原分类筛选。 |
| `use_case` | 可选，兼容原用途筛选。 |

返回：

```json
{
  "items": [],
  "total": 804,
  "limit": 24,
  "cursor": "0",
  "next_cursor": "24",
  "has_more": true,
  "index_version": "github_evolinkai_gpt_image_cases:github-xxxx"
}
```

排序沿用现有模板排序：

```text
quality_score desc -> category -> title -> case_id
```

这样案例顺序稳定，分页游标可以使用简单 offset。

### 4.3 旧接口保留

旧接口继续存在：

```text
GET /api/v2/templates?limit=1000
```

它仍返回：

```json
{ "templates": [] }
```

用途：

- 兼容已有测试。
- 兼容内部临时脚本。
- 降低本次优化的破坏面。

前端默认浏览不再使用它。

## 5. 缩略图设计

### 5.1 缩略图分档

新增两档缩略图：

| 变体 | 最大尺寸 | WebP quality | 用途 |
| --- | --- | --- | --- |
| `grid` | `720 x 900` | `84` | 案例卡片、列表浏览 |
| `preview` | `1280 x 1600` | `88` | 点击案例后的大图预览 |

旧路径：

```text
/api/v2/case-thumbnails/images/foo/output.jpg
```

自动等同：

```text
/api/v2/case-thumbnails/grid/images/foo/output.jpg
```

新增路径：

```text
/api/v2/case-thumbnails/grid/images/foo/output.jpg
/api/v2/case-thumbnails/preview/images/foo/output.jpg
```

### 5.2 缓存策略

缩略图缓存 key 必须包含：

- 当前 `index_version`
- 原始 `asset_path`
- 缩略图变体 `grid` / `preview`
- 缩略图规格版本

这样可以和旧 `360 x 450` 缓存并存，不需要部署时清空生产缓存。

响应继续使用：

```text
Cache-Control: public, max-age=31536000, immutable
```

该缓存只适用于公开案例库图片，不适用于用户生成图片、账户历史、账单、登录态或任何私人数据。

## 6. 前端改造

### 6.1 初始化

桌面端初始化：

1. 并行请求健康检查、provider、能力、历史、模型设置等现有接口。
2. 并行请求 `/templates/index` 和 `/templates/page?limit=24`。
3. 首屏只渲染第一页案例。

移动端初始化：

1. 同样请求 `/templates/index`。
2. 第一页使用 `limit=16`。

### 6.2 筛选

点击标签：

1. 更新 `activeCaseFacet`。
2. 清空当前案例列表。
3. 请求 `/templates/page?limit=pageSize&facet=...`。
4. 标签数量来自 `/templates/index`，不依赖前端全量案例。

### 6.3 加载更多

默认浏览模式下：

- 如果 `has_more=true`，显示“加载更多案例”。
- 点击后使用 `next_cursor` 请求下一页。
- 新案例追加到已有列表。

搜索模式下：

- 保留现有 `/prompt-cases/search`。
- 前端仍可使用本地渲染上限逐批显示搜索结果。
- 搜索不是本章优化重点，后续如需要再单独做搜索分页。

### 6.4 图片地址

案例卡片使用：

```text
/api/v2/case-thumbnails/grid/{asset_path}
```

大图预览使用：

```text
/api/v2/case-thumbnails/preview/{asset_path}
```

若缩略图失败，保持现有 fallback 逻辑，显示 Case 占位。

## 7. 测试计划

### 7.1 后端测试

- `/api/v2/templates/index` 返回总数、索引版本和 facet 统计。
- `/api/v2/templates/page` 支持分页、`next_cursor`、`has_more`。
- `/api/v2/templates/page` 支持 facet 筛选。
- 旧 `/api/v2/templates` 保持兼容。
- 旧缩略图路径返回 `grid` 规格。
- `grid` 缩略图最大尺寸不超过 `720 x 900`。
- `preview` 缩略图最大尺寸不超过 `1280 x 1600`。
- SVG、损坏图片等不可缩略资源仍返回 404。

### 7.2 前端静态检查

- `node --check src_skeleton/app/static/app.js`
- `node --check src_skeleton/app/mobile_static/mobile.js`

### 7.3 本地联调

检查浏览器网络：

- V2 首次打开不再出现 `/templates?limit=1000`。
- 首次案例分页请求为 `/templates/page?limit=24` 或 `/templates/page?limit=16`。
- 点击“加载更多案例”才请求下一页。
- 卡片图请求命中 `/case-thumbnails/grid/...`。
- 大图预览请求命中 `/case-thumbnails/preview/...`。

## 8. 验收标准

1. V2 首屏加载明显变轻。
2. 案例缩略图比旧版清晰，能看清主要细节。
3. 案例筛选、选择模板、取消模板仍正常。
4. V2 生成逻辑不受影响。
5. V1、Veyra 账户、登录、扣费、历史权限不受影响。
6. 所有新增/修改测试通过。
