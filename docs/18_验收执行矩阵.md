# 18. 验收执行矩阵

## 1. 验收目标

本矩阵把 `docs/13_测试验收标准.md` 落成可执行命令和边界。MVP 通过标准：

- mock 环境下全量测试通过。
- HTTP 冒烟测试覆盖会话、生图、炼金、provider、视频未配置。
- 模拟测试覆盖批量生成、幂等、版本树、provider fallback、安全拒绝。
- live-readiness 检查通过后，才能手动执行真实 API 调用。

## 2. 全量测试

```powershell
cd custom_media_agent_docs\src_skeleton
python -m pytest ..\tests -q
```

期望：

- provider contract tests 通过。
- image workflow integration tests 通过。
- video placeholder tests 通过。
- 不需要任何真实 API Key。

## 3. 模拟测试

模拟测试固定使用：

```powershell
MEDIA_AGENT_MODE=mock
MOCK_IMAGE_PROVIDER_ENABLED=true
```

覆盖：

- 纯文本生成 4 张竖版咖啡海报。
- 图片参考素材被记录到 prompt plan。
- PDF/素材引用在 request 中保留，mock 不伪造解析结果。
- 炼金生成父子版本节点。
- 默认 OpenAI 未配置时回退到 `mock_image`。
- 显式指定未配置 live provider 时返回 `provider_not_configured`。

## 4. 冒烟测试

启动服务：

```powershell
uvicorn app.main:app --host 127.0.0.1 --port 8000
```

检查：

```powershell
Invoke-RestMethod http://127.0.0.1:8000/healthz
Invoke-RestMethod http://127.0.0.1:8000/v1/providers
```

再调用：

- `POST /v1/sessions`
- `POST /v1/image/jobs`
- `GET /v1/image/jobs/{job_id}`
- `POST /v1/image/jobs/{job_id}/revise`
- `POST /v1/video/jobs`

## 5. Live-readiness 检查

真实 provider 调用不是默认 CI 的一部分。执行前必须确认：

- `.env` 或 Secret Manager 中存在对应 API Key。
- `MEDIA_AGENT_MODE=live`。
- 请求 prompt 不触发安全拒绝。
- 任务数量、质量、尺寸和预算已确认。
- 输出转存路径可写。
- 日志脱敏检查通过。

若任一条件不满足，只能执行 mock 验收，不执行真实供应商调用。

## 6. 失败处理

- 测试失败：先修复失败路径，不进入下一阶段。
- provider 未配置：记录为环境状态，不算代码失败。
- OpenAI/Gemini/Seedance SDK 缺失：应用仍应能启动并跑 mock 测试。
- 真实 API 调用失败：记录 provider、model、错误码、trace_id，不记录完整 base64 或密钥。
