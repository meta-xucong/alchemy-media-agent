# 13. 测试验收标准

## 1. 测试分层

| 层级 | 内容 |
|---|---|
| 单元测试 | prompt plan parser、provider adapter、状态机、成本估算 |
| 集成测试 | OpenAI/Gemini mock、对象存储、队列、DB、SSE |
| Agent 测试 | 意图识别、handoff、结构化输出、失败回退 |
| E2E 测试 | 上传素材 → 生图 → 炼金 → 下载 |
| 视觉回归 | 固定 prompt 的输出评分、尺寸、格式、OCR/文字检查 |
| 安全测试 | prompt 注入、敏感内容、API Key 泄露、越权下载 |
| 压测 | 批量任务、队列积压、供应商 429、重试风暴 |

## 2. MVP 验收用例

### 用例 1：纯文本生成

输入：“生成 4 张竖版咖啡新品海报，日系清爽风格。”

验收：

- 创建 4 个输出。
- 尺寸为竖版。
- 数据库记录 prompt plan、provider、model、cost。
- UI 可下载。

### 用例 2：图片参考

上传一张品牌视觉图，输入：“保持这个配色和高级感，生成电商主图。”

验收：

- Asset ready。
- Material Brief 包含配色/风格。
- 输出遵守配色。
- 版本树可见。

### 用例 3：PDF 品牌手册

上传 PDF，输入：“按品牌手册生成 6 张社媒图。”

验收：

- PDF 文本和页面摘要可见。
- Prompt Plan 引用品牌约束。
- 批量输出 6 张。

### 用例 4：炼金

选择一张图，输入：“构图不变，把背景改成冬天，文字更清楚。”

验收：

- 创建新版本节点。
- PromptPatch 包含 preserve/change。
- 新图与旧图形成父子关系。

### 用例 5：供应商切换

项目设置默认图片 provider 从 OpenAI 切到 Gemini。

验收：

- ProviderRegistry 返回正确模型。
- 新任务调用新 provider。
- 旧任务记录不被覆盖。

### 用例 6：视频未配置

调用视频生成接口但未配置 provider。

验收：

- 返回 `provider_not_configured`。
- 保存 VideoJobRequest。
- UI 显示实验性提示，不崩溃。

## 3. Agent 评测集

建立 `eval_cases/*.jsonl`：

- 意图识别：生图/改图/批量/视频/闲聊/澄清。
- 素材引用：用户说“用第一张”“不要用 PDF 第三页”。
- 安全：受保护 IP、真人隐私、违法内容。
- 提示词质量：是否包含主体、场景、风格、构图、输出参数。
- 炼金：是否正确 preserve/change。

## 4. Provider contract tests

每个 provider adapter 必须通过：

- `capabilities()` 返回完整。
- mock generate 返回标准 `ImageGenerationResult`。
- 错误码映射正确。
- timeout/retry 可控。
- 不把 base64 大字段写入日志。
- 不泄露 API Key。

## 5. 性能验收

| 指标 | MVP 目标 |
|---|---|
| API 创建任务延迟 | < 500ms，不含模型生成 |
| 素材小文件解析 | < 10s |
| SSE 状态推送延迟 | < 2s |
| 批量 10 图并发 | 可完成且状态准确 |
| provider 429 | 不触发无限重试 |

## 6. 发布检查清单

- [ ] `.env` 无真实密钥。
- [ ] 所有下载 URL 有过期时间。
- [ ] 失败任务可重试。
- [ ] 成本记录可按项目汇总。
- [ ] 安全拒绝路径有友好文案。
- [ ] OpenAPI 与后端实现一致。
- [ ] 关键 agent prompt 有版本号。
- [ ] Provider mock 可跑 CI。
