# V2 提示词转换前端闭环开发文档

## 背景

V2 后端已具备 `prompt_transform` 层：在 `ImagePromptPlan` 创建后、调用图像 provider 前，将最终提示词写入 `user_variables["generation_prompt"]`，并将转换信息写入 `user_variables["prompt_transform"]`。

上一轮审计确认后端行为可用，但桌面端和 H5 前端仍存在三个缺口：

1. 用户无法显式选择提示词转换模式，只能依赖后端按 V2 mode 自动映射。
2. 结果区没有展示转换状态、保真模式和提取出的硬约束。
3. 结果灯箱里的完整提示词没有优先展示真正送给 provider 的 `generation_prompt`。

## 目标

本轮只补 V2 前端闭环，不新增 Prompt Transform 架构层。

- 桌面端和 H5 都提供相同的提示词转换模式控制。
- 前端可将模式传入 V2 creative run。
- 后端 creative run 将模式写入 `ImagePromptPlan.user_variables["prompt_transform_mode"]`。
- provider 参数不携带 `prompt_transform_mode`，避免污染图像 provider 请求。
- V2 结果区展示转换状态与硬约束摘要。
- 灯箱和历史提示词优先显示最终 `generation_prompt`。

## 模式定义

前端暴露四个选项：

| 前端值 | 后端含义 | 行为 |
| --- | --- | --- |
| `auto` | 不显式传值 | 后端按 V2 mode 映射；手选模板走 `stable/original`，智能增强/修图/批量走 `enhanced/strict`，不会自动进入探索 |
| `stable` | original | 原样保留 Claude 输出，不包 guard；弱介入 |
| `enhanced` | strict | 提取硬约束并包裹保真 guard；中介入，保真守护最强 |
| `exploration` | off | 不包 guard，让 Claude 主动给出差异化创意路径；强创意介入 |

前端顺序固定为：`自动 / 原样 / 增强 / 探索`。后三档按创意介入强度从弱到强排列；其中 `enhanced` 的保真守护强度高于 `exploration`，但创意发散强度低于 `exploration`。

后端已有兼容：`strict/original/off` alias 仍可被 `mode.py` 接受。

## 交互设计

### 桌面端

位置：V2 生图区右侧控制列，放在画幅选择之后、清空模板之前。

设计理由：

- 这里已有数量、画幅、模板微调，是生成前控制项的自然位置。
- 使用 segmented control，符合当前模型/画幅选择的交互语言。
- 不占据主 prompt 面积，避免让工具界面变成说明页。

### H5

位置同样在 V2 控制列中，移动端纵向堆叠。

设计要求：

- 四个选项可横向滚动或自动换行，不压缩文字。
- 不遮挡生成按钮、素材上传、历史等主要工作流。
- 状态展示保持紧凑，避免移动首屏过长。

## 展示规则

结果区 `组合提示词` 增加 `Prompt Transform` 摘要：

- `Prompt Transform: 自动 / 保真增强 / 原样保留 / 关闭`
- `Fidelity: strict / original / off`
- `Applied: 是/否`
- `Constraints: N`
- 展示最多 6 条硬约束。

灯箱完整提示词：

1. 原始提示词：优先 `user_variables.user_prompt`。
2. Agent 最终提示词：优先 `user_variables.generation_prompt`。
3. 若存在 `prompt_transform`，追加转换摘要。

历史：

- `item.prompt` 已由后端写入最终提示词。
- `item.metadata.prompt_transform` 存在时，追加转换摘要。

## 验收标准

### 后端

- `/api/v2/creative/runs` 的 `output.prompt_transform_mode` 可控制转换行为。
- `prompt_transform_mode` 不进入 `prompt_plan.provider_parameters`。
- `enhanced` 会生成 strict guard。
- `stable` 在 `smart_enhance` 下也可强制 original。
- `exploration` 会关闭转换。

### 前端

- 桌面端和 H5 都能选择四种模式。
- 请求 payload 正确携带非 auto 模式。
- 结果区能展示转换状态和硬约束。
- 灯箱显示 provider 实际使用的最终提示词。
- 布局在桌面和手机宽度下不溢出、不重叠、不遮挡核心操作。

### 测试

- 后端 focused tests 覆盖 creative run 显式模式。
- JS 语法检查通过。
- 本地 HTTP smoke 验证 payload 到结果 metadata 的完整链路。
- 浏览器桌面和移动视口手动/自动 QA 通过。
