# 17. 开发前准备与本地运行手册

## 1. 范围

本手册只补齐开发执行所需准备，不扩大既有产品边界：

- MVP 完整落地生图链路。
- 视频模块只保留 `VideoGenerationAgent` / `VideoProvider` 异步任务接口。
- 本地默认走 mock provider，方便无真实 API Key 时完成开发、CI、模拟和冒烟测试。
- 真实 OpenAI/Gemini/Seedance/Veo 调用必须显式配置 API Key，并通过 live-readiness 检查后再执行。

## 2. 本地环境

- Python 3.12+
- FastAPI + Pydantic v2
- pytest
- 可选：OpenAI SDK、OpenAI Agents SDK、Google/BytePlus SDK

推荐从 `src_skeleton` 目录运行：

```powershell
python -m pytest ..\tests -q
uvicorn app.main:app --reload
```

## 3. 配置文件

复制 `src_skeleton/.env.example` 为 `.env`，本地开发不要写入真实密钥到仓库。

关键配置：

- `MEDIA_AGENT_MODE=mock`：默认，不调用真实供应商。
- `MEDIA_AGENT_MODE=live`：允许真实 provider 调用，但仍需对应 API Key。
- `DEFAULT_IMAGE_PROVIDER=openai_gpt_image`：生产默认供应商。
- `MOCK_IMAGE_PROVIDER_ENABLED=true`：默认启用 mock fallback。
- `DEFAULT_VIDEO_PROVIDER=seedance`：视频接口预留供应商。

## 4. Provider 策略

Provider 选择遵循现有文档规则：

1. 用户显式指定 provider 时优先检查该 provider。
2. provider 未配置 API Key 时返回 `provider_not_configured`，不得静默绕过安全拒绝。
3. 未显式指定 provider 且本地启用 mock 时，可回退到 `mock_image` 以完成本地测试。
4. 所有 provider 能力必须可查询，且不能因缺少第三方 SDK 或 API Key 导致应用无法启动。
5. 供应商响应中的大 base64 字段不进入日志或数据库，只允许写入受控存储。

## 5. 本地存储

MVP 使用本地存储替代对象存储：

```text
src_skeleton/.media_storage/outputs/{job_id}/{output_id}.{ext}
src_skeleton/.media_storage/thumbs/{job_id}/{output_id}.webp
```

本地 URI 仅用于开发验收。生产必须替换为私有对象存储和短期签名下载 URL。

## 6. 开发顺序

1. Provider contract：能力查询、mock generate/edit、错误映射、密钥脱敏。
2. Image job API：创建、查询、幂等、批量输出、成本记录。
3. Revision API：基于选中 output 创建新版本节点。
4. Video job API：保存请求，未配置 provider 时返回实验性友好错误。
5. Smoke test：从 HTTP 层跑通创建会话、创建生图、炼金、视频未配置。

## 7. 不允许越界

- 不把视频生成做成同步长请求。
- 不把真实 API Key 写入文档、测试、日志或代码。
- 不绕过供应商安全策略。
- 不承诺图片一次生成 100% 命中。
- 不把 Claude Code 放进高频生产主链路，除非策略开关和隔离已经准备好。
