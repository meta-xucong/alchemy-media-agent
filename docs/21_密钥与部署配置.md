# 密钥与部署配置

本项目不把真实 API Key 写进 Git 仓库。即使仓库是 private，也应把密钥放在本机环境变量、GitHub Secrets、企业 Secret Manager 或加密文件中。

## 本机部署

在新机器克隆仓库后，可以用脚本生成 `src_skeleton/.env`：

```powershell
cd custom_media_agent_docs
.\scripts\bootstrap_env.ps1
```

脚本会优先读取当前环境变量：

- `OPENAI_API_KEY`
- `ANTHROPIC_AUTH_TOKEN`

如果环境变量不存在，脚本会尝试读取：

- `%USERPROFILE%\.codex\auth.json` 中的 `OPENAI_API_KEY`
- `%USERPROFILE%\.claude\settings.json` 中的 `ANTHROPIC_AUTH_TOKEN`

已有 `.env` 时脚本不会覆盖；确认要覆盖时使用：

```powershell
.\scripts\bootstrap_env.ps1 -Force
```

## GitHub 部署

真实密钥应写入 GitHub Repository Secrets：

- `OPENAI_API_KEY`
- `ANTHROPIC_AUTH_TOKEN`
- `VPS_HOST`
- `VPS_USER`
- `VPS_SSH_KEY`
- `VPS_PORT`，可选，默认 `22`

非敏感运行参数可写入 GitHub Repository Variables：

- `OPENAI_BASE_URL`
- `ANTHROPIC_BASE_URL`
- `DEFAULT_IMAGE_PROVIDER`
- `OPENAI_IMAGE_MODEL`
- `DEFAULT_LLM_PROVIDER`
- `KIMI_LLM_MODEL`
- `IMAGE_WORK_INTENSITY`

GitHub Secrets 不能被克隆后的本地机器直接读取，这是 GitHub 的安全设计。若需要“克隆后无需手动输入密钥”的跨机器部署，应使用团队级 Secret Manager，或提交一个加密的 `.env.enc` 并通过单独渠道分发解密口令。

## VPS 一步部署

推荐使用仓库内置的 GitHub Actions 工作流：

1. 在 GitHub 仓库 `Settings -> Secrets and variables -> Actions` 中补齐 VPS 信息：

```powershell
gh secret set VPS_HOST --repo meta-xucong/alchemy-media-agent --body "your.vps.ip"
gh secret set VPS_USER --repo meta-xucong/alchemy-media-agent --body "root"
gh secret set VPS_PORT --repo meta-xucong/alchemy-media-agent --body "22"
gh secret set VPS_SSH_KEY --repo meta-xucong/alchemy-media-agent < C:\path\to\private_key
```

2. 打开 GitHub 仓库的 `Actions -> Deploy VPS -> Run workflow`。
3. 保持默认参数即可部署到 `/opt/alchemy-media-agent`，默认端口为 `8017`。

工作流会自动完成：

- 同步当前仓库代码到 VPS。
- 把 GitHub Secrets/Variables 渲染为 VPS 上的 `src_skeleton/.env`。
- 安装/检查 Docker 与 Compose。
- 构建镜像并启动 `alchemy-media-agent` 容器。
- 健康检查 `/v1/providers`。

部署完成后访问：

```text
http://<VPS_IP>:8017/
```

如果你已经 SSH 到 VPS，也可以在服务器上运行：

```bash
export GITHUB_TOKEN="github_pat_or_token_with_repo_read"
export OPENAI_API_KEY="..."
export ANTHROPIC_AUTH_TOKEN="..."
export DEPLOY_DIR="/opt/alchemy-media-agent"
export APP_PORT="8017"
git clone "https://x-access-token:${GITHUB_TOKEN}@github.com/meta-xucong/alchemy-media-agent.git" "$DEPLOY_DIR"
cd "$DEPLOY_DIR"
./scripts/deploy_vps.sh --local
```
