# 密钥与部署配置

本项目不把真实 API Key 写进 Git 仓库。即使仓库是 private，也应把密钥放在本机环境变量、GitHub Secrets、企业 Secret Manager 或加密文件中。

## 本机部署

在新机器克隆仓库后，可以用脚本生成 `src_skeleton/.env`：

```powershell
cd custom_media_agent_docs
.\scripts\bootstrap_env.ps1
```

脚本会优先读取当前环境变量：

- `OPENAI_API_KEY`
- `ANTHROPIC_AUTH_TOKEN`

如果环境变量不存在，脚本会尝试读取：

- `%USERPROFILE%\.codex\auth.json` 中的 `OPENAI_API_KEY`
- `%USERPROFILE%\.claude\settings.json` 中的 `ANTHROPIC_AUTH_TOKEN`

已有 `.env` 时脚本不会覆盖；确认要覆盖时使用：

```powershell
.\scripts\bootstrap_env.ps1 -Force
```

## GitHub 部署

真实密钥应写入 GitHub Repository Secrets：

- `OPENAI_API_KEY`
- `ANTHROPIC_AUTH_TOKEN`

非敏感运行参数可写入 GitHub Repository Variables：

- `OPENAI_BASE_URL`
- `ANTHROPIC_BASE_URL`
- `DEFAULT_IMAGE_PROVIDER`
- `OPENAI_IMAGE_MODEL`
- `DEFAULT_LLM_PROVIDER`
- `KIMI_LLM_MODEL`
- `IMAGE_WORK_INTENSITY`

GitHub Secrets 不能被克隆后的本地机器直接读取，这是 GitHub 的安全设计。若需要“克隆后无需手动输入密钥”的跨机器部署，应使用团队级 Secret Manager，或提交一个加密的 `.env.enc` 并通过单独渠道分发解密口令。
