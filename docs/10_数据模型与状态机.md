# 10. 数据模型与状态机

## 1. 核心实体

```text
User -> Organization -> Project -> Session -> Message
Project -> Asset -> AssetDerivative
Session -> GenerationJob -> GenerationOutput
GenerationOutput -> RevisionBranch
GenerationJob -> ModelInvocation -> CostRecord
```

## 3. 状态机

### Asset 状态机

```text
uploaded -> scanning -> stored -> extracting -> analyzed -> ready
   |           |            |             |           |
   -> rejected -> failed     -> failed      -> failed    -> archived
```

### Image Job 状态机

```text
created -> queued -> planning -> safety_check -> generating -> postprocessing -> evaluating -> ready
               |        |             |             |                |             |
               -> canceled           -> rejected     -> failed         -> failed      -> failed
```

### Video Job 状态机

```text
created -> queued -> submitted -> processing -> downloading -> transcoding -> ready
               |          |            |             |             |
               -> canceled -> submit_failed -> provider_failed -> expired -> failed
```

## 4. 对象存储路径

```text
/{tenant_id}/{project_id}/assets/{asset_id}/original
/{tenant_id}/{project_id}/assets/{asset_id}/derivatives/{name}
/{tenant_id}/{project_id}/generations/{job_id}/{output_id}.{ext}
/{tenant_id}/{project_id}/generations/{job_id}/thumb_{output_id}.webp
```

## 5. 版本树

版本关系不靠文件名，必须写表：

```text
generation_outputs.version_parent_id
```

可支持：

- 回退到某个版本。
- 比较两个版本 prompt patch。
- 复用某个版本作为参考图。
- 批量导出某个分支。

## 6. 幂等与重试

- `idempotency_key = hash(session_id + user_message_id + normalized_prompt_plan + asset_ids + request_index)`。
- 重试同一个任务不产生重复收费记录；如果 provider 实际已扣费但返回失败，要记录 `unknown` 状态并人工/后台对账。
