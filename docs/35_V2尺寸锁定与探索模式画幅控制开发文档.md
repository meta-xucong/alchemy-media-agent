# V2 尺寸锁定与探索模式画幅控制开发文档

## 背景

V2 提示词增强模块包含自动、原样、增强、探索四种前端档位。探索档会让 Claude Code 主动改变创意路径，例如角度、姿态、光影、背景层次、裁切和色彩氛围。这个行为可以改变画面观感，但不应该在用户已经手动选择画幅时改变实际输出尺寸。

当前前端的“默认”画幅表示自动画幅，不会向 V2 后端传递 `aspect_ratio`。因此默认画幅下，Provider 或 Claude Code 可以根据模板、提示词和模型默认行为决定最终尺寸。该行为需要在前端说明清楚。

## 设计原则

1. 自动画幅允许变化
   用户选择“自动”时，不传 `aspect_ratio` 或 `size`。输出尺寸可能随模板、Provider 默认值、Claude 创意策略或模型能力变化。

2. 手动画幅必须锁定
   用户选择竖版、方图、横版或后续明确尺寸时，该选择是硬生成参数。自动、原样、增强、探索都不能覆盖它。

3. 探索只改变创意路径
   探索可以改变视角、裁切、背景层次、光影、色彩和场景调度，但必须服从用户手动选择的画幅。

4. 比例兜底只在手动画幅时追加
   当用户显式选择画幅时，系统在最终提示词中追加短比例要求，作为 Provider 不严格执行尺寸参数时的兜底。自动画幅不追加比例要求。

5. 不移植复杂尺寸面板
   参考 `kadevin/ilab-gpt-conjure` 的尺寸硬约束思想，但 V2 保持小白友好，只保留自动、竖版、方图、横版四个入口。

## 前端行为

- “默认”改名为“自动”，并通过 tooltip/提示文字说明“自动画幅可能随模式或 Provider 变化”。
- 竖版、方图、横版按钮继续传递 `1024x1536`、`1024x1024`、`1536x1024`。
- V2 提交预览、运行提示和 prompt plan 展示中应能区分：
  - `Aspect: auto`
  - `Aspect: 1024x1536 (locked)`

## 后端行为

- `compose_prompt_plan()` 接收用户原始 `requested_output`，只根据该对象判断是否有手动画幅锁。
- `provider_parameters` 最终仍由用户输出覆盖 Claude 输出。
- 如果 `requested_output.aspect_ratio` 或 `requested_output.size` 为非空、非 `auto`、非 `default`：
  - 写入 `user_variables["aspect_lock"]`
  - 确保 `provider_parameters["aspect_ratio"]` 或 `provider_parameters["size"]` 保留用户值
  - 在 prompt 尾部追加 `Required output aspect ratio: W:H.`
- 如果用户未显式选择画幅：
  - 不追加比例兜底
  - 写入 `user_variables["aspect_lock"]` 为自动状态

## 验收标准

- 手动横版 + 探索：最终 `provider_parameters.aspect_ratio` 保持 `1536x1024`。
- 手动横版 + 探索：最终 prompt 包含 `Required output aspect ratio: 3:2.`。
- 自动画幅 + 探索：最终 `provider_parameters` 不应凭空出现用户锁定字段。
- 自动画幅 + 探索：最终 prompt 不追加比例兜底。
- 桌面和 H5 前端都显示“自动”而不是“默认”，并有画幅含义说明。
- 前端 JS 语法检查通过。
- V2 后端相关 pytest 通过。
