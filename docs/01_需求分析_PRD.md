# 01. 需求分析 / PRD

## 1. 产品定位

建设一个类似 ChatGPT 交互体验的“定制化生成图片和视频”Agent 平台。用户通过对话输入提示词、上传参考素材，并在多轮对话中持续调整，最终获得可下载、可复用、可批量生成的图片或视频资产。

本阶段聚焦“生图 Agent”，视频生成模块保留接口、任务状态机和供应商适配框架。

## 2. 目标用户

- 内容创作者：社媒封面、广告素材、海报、电商主图、IP 角色、漫画分镜。
- 品牌/市场团队：根据品牌手册、Campaign brief、参考图批量出图。
- 设计/运营团队：快速生成多版本、对比、修改、导出。
- 内部研发/模型团队：评估不同模型、供应商、提示词模板、成本与效果。

## 3. 核心使用场景

### 3.1 首次生图

用户输入自然语言需求，例如“生成 6 张适合小红书封面的咖啡新品图，风格参考我上传的品牌手册”。系统完成素材解析、需求澄清、提示词结构化、模型选择、图片生成、结果展示。

### 3.2 上传参考素材

支持上传：

- 图片：风格、主体、构图、配色、品牌视觉、角色一致性参考。
- 短视频：如果模型支持视频理解，直接分析；否则抽帧、提取镜头语言、动作、人物、场景。
- PDF：优先保留页面视觉和文本。
- PPT/DOC：默认提取文本；如需保留图表/排版，转换成 PDF 再处理。
- 表格：适合批量生成，例如每行一个 SKU 或广告版本。

### 3.3 批量生成

用户指定数量、规格、风格变量、文案变量、SKU 列表或批量 Prompt。系统将批量任务拆成多个子任务，进入队列并行处理，结果按批次、版本和评分展示。

### 3.4 炼金 / 继续调整

用户对某张或某批结果继续输入：“把左边人物改成背影”“保持构图，换成冬季氛围”“文字更像杂志封面”。系统基于历史提示词、参考图、选中输出和用户修改要求生成 prompt patch，再调用图片编辑或多轮图片生成接口。

### 3.5 视频预留

视频生成先不做完整产品闭环，但需要预留：文生视频、图生视频、参考素材生视频、任务查询、取消、下载、失败重试、费用估算、供应商切换。

## 4. 功能范围

### MVP 必须有

| 模块 | 能力 | 说明 |
|---|---|---|
| Chat UI | 对话输入、文件上传、历史会话 | 类 ChatGPT 产品形态 |
| 素材解析 | 图片/PDF/PPT/DOC/表格基础解析 | 输出 Material Brief |
| 生图 Agent | 提示词优化、模型选择、调用 GPT Image 2 | 以 gpt-image-2 为默认图片模型 |
| 批量生成 | n 张图片、变量矩阵、任务队列 | 支持暂停/取消/重试 |
| 炼金 | 基于已有输出继续修改 | 版本树可回溯 |
| 模型切换 | OpenAI/Gemini 等 provider 配置 | API Key 方式接入 |
| 结果管理 | 画廊、下载、收藏、删除、版本对比 | 输出 PNG/JPEG/WebP |
| 审计观测 | 调用日志、成本、错误、trace | 便于排错与优化 |
| 安全风控 | 内容安全、版权提示、密钥隔离 | 默认留痕 |

### MVP 不做，但接口预留

| 模块 | 预留点 |
|---|---|
| 视频生成 | `/v1/video/jobs`、`VideoProvider`、异步任务状态机 |
| 高级资产库 | 品牌知识库、向量检索、长期记忆 |
| 协作 | 多人项目、评论、审批流 |
| 商业化 | 套餐、余额、发票、组织权限 |
| 自动发布 | 小红书/抖音/Shopify 等渠道发布 |

## 5. 关键用户流程

1. 用户创建会话或项目。
2. 输入需求并上传素材。
3. 系统解析素材并生成 Material Brief。
4. 中枢调度 Agent 判断是否需要澄清；如果需求足够明确，进入生图规划。
5. Prompt Architect 生成结构化提示词和参数。
6. Provider Selector 选择图片模型与成本/质量档位。
7. Image Generation Agent 调用图片模型，返回候选图。
8. Critic Agent 对结果进行自动检查：是否符合尺寸、主体、文字、风格、安全规则。
9. UI 展示结果，用户下载或继续调整。
10. 如果继续调整，Revision Agent 基于选中输出和修改要求生成 prompt patch，进入编辑/再生成。

## 6. 非功能要求

| 类别 | 要求 |
|---|---|
| 可用性 | MVP 单区部署 99%；后续多区/多供应商降级 |
| 性能 | 对话响应可流式返回；图片任务走异步队列；批量任务实时进度 |
| 成本 | 每次调用记录 provider、model、参数、输入输出、估算成本 |
| 可观测 | trace_id 串联用户请求、agent handoff、模型调用、存储写入 |
| 安全 | API Key 加密存储；租户隔离；文件扫描；敏感操作审计 |
| 合规 | 用户声明素材权利；禁止明显侵权/违法/高风险内容；保留 provenance 信号 |
| 可扩展 | 图片/视频/LLM 均通过 provider registry 切换 |
| 可测试 | provider mock、golden prompt、视觉回归、E2E 回放 |

## 7. 成功指标

- 首次生成完成率 ≥ 95%。
- 图片生成成功率 ≥ 97%，供应商错误可重试。
- 用户一轮炼金后满意率 ≥ 60%。
- 平均每个任务可追踪到完整 prompt、素材、模型、输出和成本。
- 新增一个图片 provider 的工程量控制在 1-2 个工作日内。

## 8. 边界与风险

- 开源 agent 框架免费，不代表模型调用免费。
- Claude Code 作为线上中枢会增加成本、权限、延迟和稳定性复杂度，建议以策略开关决定是否进入高频主路径。
- 图片生成结果不完全确定，需要版本化、评分、人工选择，不应承诺 100% 一次命中。
- PDF/PPT/DOC 的视觉元素处理方式不同，PPT/DOC 如需保留视觉，应转换为 PDF 或页面图片后分析。
