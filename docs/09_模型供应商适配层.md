# 09. 模型供应商适配层

## 1. 目标

所有 LLM、图片、视频模型都通过可插拔 provider 接入，支持 API Key 配置、模型切换、能力探测、成本估算、重试熔断和审计。

## 2. Provider 类型

```text
LLMProvider
ImageProvider
VideoProvider
EmbeddingProvider optional
ModerationProvider optional
```

## 3. 统一配置

`.env.example`：

```bash
OPENAI_API_KEY=
ANTHROPIC_API_KEY=
GOOGLE_API_KEY=
BYTEPLUS_API_KEY=
DEFAULT_LLM_PROVIDER=openai
DEFAULT_IMAGE_PROVIDER=openai_gpt_image
DEFAULT_IMAGE_MODEL=gpt-image-2
DEFAULT_VIDEO_PROVIDER=seedance
ORCHESTRATION_MODE=runtime_first
```

生产环境不要只靠 `.env`：

- 本地开发：`.env`。
- 预发/生产：KMS/Secret Manager/Vault。
- 多租户 BYOK：用户 key 加密后入库，调用时临时解密。

## 4. ImageProvider 接口

```python
class ImageProvider(Protocol):
    name: str
    async def capabilities(self) -> ImageCapabilities: ...
    async def generate(self, request: ImageGenerationRequest) -> ImageGenerationResult: ...
    async def edit(self, request: ImageEditRequest) -> ImageGenerationResult: ...
    async def estimate_cost(self, request: ImageGenerationRequest) -> CostEstimate: ...
```

## 7. 参数归一化

不同供应商参数不一致，内部统一字段：

| 内部字段 | OpenAI | Gemini | Video provider |
|---|---|---|---|
| `prompt` | prompt/input | prompt/contents | prompt |
| `asset_refs` | image/file_id | media parts | image/video URLs |
| `size` | size | aspect/resolution | resolution/aspect_ratio |
| `quality` | low/medium/high/auto | model-specific | model-specific |
| `format` | png/jpeg/webp | mime/output | mp4/mov |
| `count` | n | candidate count | usually 1 |
| `stream` | stream | streaming if supported | async task |

## 8. 能力探测与路由

ProviderSelector 选择逻辑：

1. 用户指定 provider/model 优先。
2. 检查 provider 是否配置 API Key。
3. 检查任务能力是否匹配：编辑、透明背景、文本渲染、尺寸、批量。
4. 检查预算、限流、地域和合规。
5. 如果默认 provider 不可用，选择 fallback。

## 9. 重试、熔断和降级

- 429/限流：指数退避，必要时换 provider。
- 5xx/超时：重试 1-2 次。
- 参数不支持：ProviderSelector 修改参数。
- 安全拦截：不自动换 provider 绕过安全。
- 成本超限：请求用户确认或降低质量/数量。

## 10. OpenAI GPT Image 2 适配建议

- 对可对话、可反复编辑体验，优先走 Responses API 的 image generation tool。
- 对单次生成/编辑，也可封装 Images API。
- 输出统一转存对象存储，不依赖临时返回。
- 保存 provider 原始 response 的安全摘要，不保存过长 base64 到数据库。

## 11. Gemini 图片适配建议

- 作为多模态参考、图像生成和编辑备选。
- 需要单独记录模型能力：是否支持视频作为输入、是否支持多图、输出分辨率、文字渲染强度。

## 12. Seedance/Veo 视频适配建议

- 一律异步。
- provider task id 与内部 job id 分离。
- 结果 URL 要立即下载/转存，避免过期。
- 记录 duration、resolution、aspect ratio、prompt、参考图、任务日志。
