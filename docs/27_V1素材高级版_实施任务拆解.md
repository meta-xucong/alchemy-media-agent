# 27. V1 素材高级版实施任务拆解

## 1. 目标

本文把 V1 素材高级版拆成可执行开发任务。开发时遵循：

- 先保证基础版完全不变。
- 再加高级版 schema 和后端计划。
- 最后逐步接真实参考图、Logo 后处理和局部编辑。

## 2. 推荐开发顺序

```text
文档确认
-> feature flag
-> schema 和校验
-> AssetPlan/PromptPlan
-> 历史记录
-> 前端高级版入口
-> mock provider 测试
-> 真实 provider 参考图
-> Logo 后处理
-> mask edit
-> 风控和观测
```

## 3. Epic V1ADV-0：开关与兼容

| ID | 任务 | 说明 | 验收 |
|---|---|---|---|
| V1ADV-001 | 增加高级版总开关 | `V1_ADVANCED_ASSETS_ENABLED` | 关闭时前端不展示高级版，API 拒绝高级版请求 |
| V1ADV-002 | 增加 `asset_mode` | `basic` 默认值 | 不传字段时现有测试全部通过 |
| V1ADV-003 | 禁止模式混用 | 后端校验 `asset_ids` 和 `asset_intents` | 混用返回 `asset_mode_conflict` |
| V1ADV-004 | 历史兼容 | 旧记录按基础版显示 | 旧历史不报错 |

## 4. Epic V1ADV-1：Schema 与校验

| ID | 任务 | 说明 | 验收 |
|---|---|---|---|
| V1ADV-101 | 定义 `AssetIntent` schema | role、priority、preservation、strength、placement、consent | 单测覆盖合法和非法输入 |
| V1ADV-102 | 定义 role 枚举 | 见文档 24 | 未知 role 被拒绝 |
| V1ADV-103 | 授权校验 | 人脸、Logo、商标强制 consent | 未授权请求无法生成 |
| V1ADV-104 | 素材状态校验 | 只允许 ready 素材进入任务 | 非 ready 返回 `asset_not_ready` |
| V1ADV-105 | 素材数量限制 | 总数、参考图数、Logo 数限制 | 超限返回明确错误 |

## 5. Epic V1ADV-2：素材处理与派生文件

| ID | 任务 | 说明 | 验收 |
|---|---|---|---|
| V1ADV-201 | 替换 mock upload | 接入本地或对象存储文件保存 | 上传后文件真实可读 |
| V1ADV-202 | 图片规范化 | 统一方向、格式、尺寸 | provider 调用使用 normalized 文件 |
| V1ADV-203 | 缩略图生成 | 列表用小图 | 前端不加载原图作为缩略图 |
| V1ADV-204 | 图片-only 限制 | 高级版只允许 `image/*` | PDF/DOC/CSV 被拒绝 |
| V1ADV-205 | MaterialBrief 增强 | 主色、构图、文字、疑似 Logo/人脸 | 高级版风格提示可用 |
| V1ADV-206 | AssetVisionProfile | 视觉模型/算法输出结构化画像 | 图片上传后可见视觉画像 |
| V1ADV-207 | provider file 缓存 | 缓存 file_id 或上传引用 | 过期后可重新上传 |

备注：高级版要承诺“参考素材生效”，就必须完成真实图片保存和 provider 图片输入；只做提示词增强不能视为完整高级版。

## 7. Epic V1ADV-4：PromptPlan

| ID | 任务 | 说明 | 验收 |
|---|---|---|---|

## 14. 第一阶段 MVP 范围

建议第一阶段只做：

- 基础参数和高级素材增强协作叠加。
- 高级版只支持图片素材。
- 高级版三类用途：风格参考、主体参考、Logo/标识。
- `AssetPlan`、`PromptPlan`。
- 历史记录完整展示。
- provider capability 声明。
- mock provider 完整测试。
- 真实 provider 验证风格参考和主体参考的图片输入通路。

暂缓：

- 真正 Logo 像素级叠加。
- 高级视觉复检自动重试。
- 人脸相似度复检。
- 局部 mask 编辑。
- 多素材复杂冲突解决。

## 15. 第二阶段增强

第二阶段做：

- Logo 后处理。
- 真实 provider 参考图完善。
- AssetVisionProfile 视觉画像。
- Kimi 基于视觉画像做 PromptPlan 规划。
- 局部编辑。
- 缩略图和素材派生文件生产化。
- 429 冷却和任务队列。

## 16. 第三阶段生产化

第三阶段做：

- VPS 持久队列。
- 对象存储替换本地临时文件。
- 审计后台。
- 视觉复检。
- 复检驱动的自动重试。
- 成本统计。
- 多 provider 优先级和冷却策略。

## 18. 开发自检清单

每次提交前检查：

- 是否影响基础版旧请求。
- 是否保存原始提示词和最终提示词。
- 是否把 provider 不支持的能力提前拦截。
- 是否把用户授权写入快照。
- 是否有 mock 测试。
- 是否有历史展示字段。
- 是否没有把真实 API key 写进仓库。
- 是否没有把用户上传原图暴露为长期公开 URL。
