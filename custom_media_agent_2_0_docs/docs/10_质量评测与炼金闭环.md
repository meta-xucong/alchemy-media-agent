# 10. 质量评测与炼金闭环

## 目标

2.0 的质量提升不是一次 prompt 完成，而是持续闭环：

```text
案例结构化 -> 召回 -> prompt 合成 -> 生图 -> 评价 -> 用户反馈 -> 排序和策略优化
```

## 评价对象

1. 案例结构化质量。
2. 召回相关性。
3. prompt plan 质量。
4. 生图输出质量。
5. 修订建议质量。
6. 用户满意度。

## 评测集

MVP 至少建立三类评测集：

1. 模板定制评测集：用户选择模板并替换主体。
2. 智能增强评测集：用户只给一句短需求。
3. 风险评测集：品牌、肖像、授权不明、禁止内容。

每类至少 30 条样例。

## 召回评测

指标：

1. top 3 命中率。
2. top 10 覆盖率。
3. 多样性。
4. 风险过滤准确率。
5. 案例解释可读性。

人工标注：

1. 高相关。
2. 部分相关。
3. 不相关。
4. 不应召回。

## Prompt 评测

指标：

1. 结构完整度。
2. 用户变量覆盖率。
3. 案例精华提炼质量。
4. 风险规避质量。
5. 供应商参数适配度。

Prompt plan 必须包含：

1. 主 prompt。
2. 可变字段。
3. 参考案例。
4. 风格理由。
5. 风险说明。

## 图片评测

自动评分：

1. 主体清晰度。
2. 构图。
3. 光影。
4. 色彩。
5. 商品可见性。
6. 文字可读性。
7. 品牌安全。

当前实现阶段：

1. `OutputReviewService` 对每个 `ImageOutput` 生成 `ImageReviewDecision`。
2. `VisualCriticAgent` 已作为 OpenAI Agents SDK 复检边界接入。
3. 未配置真实视觉模型时，复检只基于 provider metadata、prompt plan、错误状态、mock/live 标记和负向 prompt 完整度。
4. 这种模式必须在 `analysis_mode` 中标注为 `metadata_rules` 或 `sdk_agent_metadata_fallback`，不能宣称完成像素级视觉审查。
5. 接入真实视觉模型后，仍沿用同一个 `ImageReviewDecision` schema，把像素观察、风险和返工建议写回现有闭环。

人工反馈：

1. 选中。
2. 可用。
3. 接近可用。
4. 不可用。
5. 需要修订。

## 炼金闭环

用户反馈进入以下优化：

1. 案例质量权重。
2. 召回排序权重。
3. prompt atom 权重。
4. provider 选择策略。
5. 模板推荐排序。
6. 风险提示文案。

不允许自动修改：

1. provider 原始内容。
2. 授权策略。
3. 硬门禁规则。

## A/B 实验

可实验项：

1. 召回权重。
2. prompt 合成策略。
3. provider 选择策略。
4. 自动复检阈值。
5. 模板卡片排序。

不可实验项：

1. 安全阻断逻辑。
2. 用户授权要求。
3. 隐私删除要求。

## 发布门槛

每次升级 agent instructions、检索权重或安全策略前，必须跑：

1. 模板定制评测集。
2. 智能增强评测集。
3. 风险评测集。
4. 至少 10 个端到端 smoke run。

若风险评测失败，不允许发布。
