# 13. 测试验收标准

## 测试分层

1. Unit tests。
2. Contract tests。
3. Integration tests。
4. Agent evals。
5. End-to-end smoke tests。
6. Safety regression tests。

## Unit Tests

覆盖：

1. Markdown parser。
2. Prompt atom extractor。
3. Provider manifest validator。
4. Risk tag classifier。
5. Ranking scorer。
6. State machine transition。

## Contract Tests

覆盖：

1. SDK tool 输入输出 schema。
2. Domain Service 返回格式。
3. Provider adapter manifest。
4. OpenAPI schema。
5. JSON 模板。

所有 tool 必须测试：

1. 正常返回。
2. 空结果。
3. 参数错误。
4. 超时。
5. 权限不足。
6. 安全阻断。

## Integration Tests

覆盖：

1. GitHub provider 同步到 PromptCase。
2. PromptCase 入库和检索。
3. Agent 调用 search tool。
4. Agent 生成 prompt plan。
5. SafetyService 阻断高风险 prompt。
6. GenerationService 创建 mock job。

## Agent Evals

评测用例至少包含：

1. 电商商品主图。
2. 广告海报。
3. 人像风格。
4. UI 概念图。
5. 角色设定。
6. 对比图。
7. 授权不明案例。
8. 用户要求复刻品牌或名人。

评分项：

1. 模式判断是否正确。
2. 案例召回是否相关。
3. Prompt 是否完整。
4. 风险处理是否正确。
5. 输出结构是否符合 schema。

## E2E Smoke

上线前必须通过：

1. `/api/v2/health`。
2. provider sync。
3. case search。
4. template customize run。
5. smart enhance run。
6. image job mock 或真实供应商。
7. output feedback。
8. trace 查询。

## Veyra 账号与扣费验收

sub2api 侧必须通过：

1. `GET /api/veyra/portal/config` 返回配置的 `alchemy_base_url`。
2. 未登录从 Veyra 首页点击“聚合平台”会进入 sub2api 登录页，登录后进入 `/dashboard`。
3. 未登录从 Veyra 首页点击“Alchemy”会进入 sub2api 登录页，登录后回到 `/_veyra/return?target=alchemy` 并继续跳到 Alchemy。
4. 已登录点击“Alchemy”会调用 `POST /api/veyra/login-ticket`，并带 `Authorization: Bearer <auth_token>`。
5. ticket 只能交换一次，过期或复用必须失败。
6. `/api/*`、`/v1/*`、`/v1beta/*` 不被 portal middleware 拦截。

Alchemy 侧必须通过：

1. 桌面和 H5 都能消费 `?ticket=...`。
2. `POST /api/v2/veyra/login` 返回本地 Alchemy session token。
3. 后续 V2 请求自动带 `Authorization: Bearer <alchemy_veyra_access_token>`。
4. `GET /api/v2/veyra/me` 展示 sub2api 余额。
5. `GET /api/v2/veyra/history` 只返回当前用户的 V2 生成记录。
6. `GET /api/v2/veyra/usage` 只返回当前用户的 Alchemy 扣费记录。
7. `POST /api/v2/creative/runs/async` 在 `VEYRA_AUTH_ENABLED=true` 时必须从 Authorization 解析用户，不能信任请求体里的 `veyra_user_id`。
8. 余额不足时生成任务失败，且 image provider 没有被调用。

隔离检查必须通过：

```powershell
rg -n "/api/v1|/v1/|custom_media_agent_docs|legacy_image_bridge|alchemy_1_0_bridge" app
```

在 `custom_media_agent_2_0` 后端代码中不得出现 V2 依赖 V1 后端的路径或桥接标记。

## 安全回归

安全测试用例必须包含：

1. 未授权名人肖像。
2. 第三方品牌 logo 复刻。
3. 受版权保护角色。
4. 用户上传身份证或隐私信息。
5. 明确违法内容。
6. 案例授权不明但用户要求商用。

任何安全回归失败，不能发布。

## 性能验收

MVP 指标：

1. 案例搜索 P95 小于 800ms。
2. Agent 创建运行首包 P95 小于 6s。
3. 生图 job 创建 P95 小于 2s。
4. Provider 无变化同步检查小于 30s。
5. 1,000 个案例以内检索可稳定运行。

## 隔离验收

必须验证：

1. 关闭 v2 backend 后旧版 API 正常。
2. 关闭旧版 backend 后 v2 health 正常。
3. v2 Redis key 不出现在旧版 namespace。
4. v2 对象存储不写入旧版 prefix。
5. v2 DB migration 不修改旧表。
