# 01. PRD 与成功指标

## 用户画像

目标用户分三类：

1. 电商与广告客户：需要商品主图、广告创意、节日促销图、社媒图。
2. 品牌与内容团队：需要高审美、高一致性的视觉方向。
3. 设计服务方：需要快速给客户出多套可比选方向，并能解释风格来源。

## 核心问题

普通生图系统常见问题：

1. 用户 prompt 太短，模型理解空间太大。
2. 输出风格不稳定，审美质量波动。
3. 用户不知道如何描述构图、光影、材质、品牌语气。
4. 相似需求重复调试，优秀案例没有被系统性复用。
5. 客户定制时，既想“像某个案例”，又不想完全复制。

2.0 的价值是把优秀案例沉淀为可解释的创意资产，并通过 agent 自动匹配、拆解、重组。

## 功能范围

### P0

1. GitHub ResourceProvider 同步。
2. PromptCase 结构化。
3. 案例模板库浏览。
4. 模板定制 prompt 生成。
5. 智能增强 prompt 生成。
6. 生图任务创建和状态查询。
7. 结果反馈与案例贡献度记录。
8. 安全与授权门禁。

### P1

1. 多 provider 接入。
2. 多模型 provider 策略。
3. 批量生成与 A/B 变体。
4. 视觉批评 agent。
5. 自动二次修订。
6. 运营精选案例集。

### P2

1. 视频生成 planner。
2. 企业私有案例库。
3. 品牌风格库。
4. 团队协作审批。
5. 多语言 prompt 市场。

## 用户流程

### 模板定制

1. 用户进入 2.0 前端。
2. 浏览或搜索模板案例。
3. 选择模板。
4. 输入自己的商品、品牌、文案、尺寸、用途、限制。
5. 中枢 agent 调用模板指导能力，生成 prompt plan。
6. 安全服务检查授权和风险。
7. 生图服务创建任务。
8. 用户查看结果、选择、下载、反馈或继续修改。

### 智能增强

1. 用户只输入一句简单需求。
2. 中枢 agent 判断意图和用途。
3. CaseStrategistAgent 生成召回策略。
4. 检索服务返回相关案例。
5. PromptComposerAgent 提炼可复用要素并组合新 prompt。
6. VisualCriticAgent 或评价服务给出预评估。
7. 生图服务执行。
8. 输出记录关联到召回案例和 prompt 组成。

## 成功指标

### 体验指标

1. 首次生成满意率：用户首次输出选择“可用”或“接近可用”的比例。
2. 二次修订收敛率：两轮内达到用户目标的比例。
3. 模板转化率：模板浏览后发起生成的比例。
4. 智能增强采用率：无模板输入时使用 agent 增强结果的比例。

### 质量指标

1. Prompt 结构完整度。
2. 案例召回相关性。
3. 风格一致性评分。
4. 构图和可读性评分。
5. 品牌安全通过率。
6. 图片供应商失败率。

### 工程指标

1. ResourceProvider 同步成功率。
2. 案例解析成功率。
3. Agent tool 调用成功率。
4. P95 任务创建延迟。
5. Trace 覆盖率。
6. 回滚隔离能力。

## 明确验收

MVP 完成时，必须能演示：

1. 从 GitHub 仓库同步案例。
2. 在前端看到模板库。
3. 选择一个模板并替换客户商品生成 prompt。
4. 输入一句简单需求并自动匹配案例生成增强 prompt。
5. 创建至少一个生图任务。
6. 在数据库或日志中追踪案例来源、agent 决策、工具调用和输出反馈。
7. 关闭 2.0 服务不影响旧版。
