# 09. 安全合规与授权风控

## 基本原则

安全、版权、商用授权、肖像、隐私和平台内容政策属于确定性硬门禁。Agent 可以建议替代方案，但不能推翻硬门禁。

## 风险分类

### 内容安全

1. 暴力、色情、仇恨、非法活动。
2. 未成年人相关高风险内容。
3. 欺骗性身份、伪造证件、危险指导。

### 授权风险

1. 第三方品牌标识。
2. 未授权商标。
3. 可识别名人或公众人物肖像。
4. 受版权保护角色。
5. 仓库案例授权不明。

### 商用风险

1. 案例仅可作为灵感。
2. 原始预览图不可作为最终资产。
3. Prompt 大段复用可能导致风格或表达过度相似。
4. 用户上传素材未确认权利。

### 隐私风险

1. 用户上传真实人脸。
2. 订单、地址、手机号、身份证等敏感信息。
3. 企业内部未公开素材。

## SafetyDecision

所有关键阶段必须生成 `SafetyDecision`：

1. 输入阶段。
2. 案例召回阶段。
3. prompt 合成阶段。
4. 生图前。
5. 输出解释前。

决策类型：

1. `allow`。
2. `allow_with_warning`。
3. `transform_required`。
4. `blocked`。
5. `need_user_confirmation`。

示例见 [safety_decision.example.json](../templates/safety_decision.example.json)。

## 案例使用策略

### 可直接用于模板

条件：

1. 授权清楚。
2. 无第三方高风险元素。
3. Prompt 可抽象为通用视觉结构。
4. 原图不作为最终交付资产。

### 仅可用于灵感

条件：

1. 授权不明。
2. 包含可能受保护人物、品牌或角色。
3. 原 prompt 强依赖具体 IP。
4. 案例来源需要进一步确认。

### 禁止使用

条件：

1. 明确不允许商业使用。
2. 涉及违法或平台禁止内容。
3. 用户请求复刻高风险个人、品牌或受保护作品。

## Prompt 合成限制

PromptComposerAgent 必须：

1. 提取抽象风格，不复制不可授权元素。
2. 避免逐字长段复用原 prompt。
3. 对品牌、人物、角色做安全替换。
4. 在输出中标明来源案例只是风格参考。
5. 给出可解释的替代描述。

## 用户授权确认

以下情况需要用户确认：

1. 使用用户上传品牌资产。
2. 使用真实人脸或肖像。
3. 生成近似某个客户提供参考图的商业图。
4. 使用企业内部资料。

确认记录必须保存到 `v2_policy_decisions` 或相关授权表。

## 审计

每个输出必须能回答：

1. 参考了哪些案例。
2. 哪些元素被复用。
3. 哪些元素被替换或规避。
4. 安全检查结果是什么。
5. 用户是否确认授权。
6. 实际调用了哪个供应商和模型。

## 默认文案口径

前端对用户解释时，应使用：

```text
此模板用于参考构图、光影和视觉表达方式。系统会根据你的主体和用途生成新的提示词，不会直接复用原图作为最终商用素材。
```

禁止使用：

```text
完全学习了这个风格。
可以无风险商用所有案例。
100% 复刻原图效果。
```
