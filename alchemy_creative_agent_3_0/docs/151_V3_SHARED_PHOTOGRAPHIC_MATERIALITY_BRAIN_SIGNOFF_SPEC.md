# Doc151 — V3 Shared Photographic Materiality Brain Sign-off

## Status

Implemented shared Human Realism refinement. This complements Doc143's shared
pixel review and Doc150's human-presence resolution; it does not replace the
Remote Brain, capability envelope, canonical prompt binding, Provider, review,
retry, or final-delivery architecture.

## Controlled observation

The Doc150 blue-dress comparison corrected the generic display smile through a
Brain-authored attention state. Its person still retained an overly uniform,
beauty-clean surface in close inspection. The signed prompt used only the
generic phrase `photorealistic detail`, which did not make the existing
`photographic_material_requirement=camera_observed_human_materiality`
meaningful in the final renderer direction.

This is a shared semantic completeness gap. It can affect any real visible
person in bright, dark, editorial, candid, lifestyle, product-on-person, or
portrait imagery. It is not a child, skin-tone, fashion, template, or provider
problem.

## Decision

When the shared Human Realism contract is active, the Remote Brain must treat
generic photographic adjectives as insufficient proof of materiality. In its
one complete canonical direction, it resolves how the person is physically
observed through the already user-owned scene's light, depth, and natural
surface response. It does this as a whole photographic judgement, alongside
expression, reference truth, complexion, and style—not as a list of skin
features.

At the existing Human Naturalness re-sign stage, a candidate that only asserts
photorealism without resolving materiality is rewritten by the same Remote
Brain. The local runtime does not inspect wording, supply a phrase, or decide
what the material treatment should be.

## Prohibited approaches

- face/skin pore, shine, wrinkle, hue, or retouch keyword lists;
- hard numerical texture thresholds or local pixel heuristics;
- child/adult/region/template-specific material recipes;
- local negative prompts, post-processing, model routing, or a special retry;
- treating a pleasant commercial look as a substitute for observed material.

## Shared validation boundary

Doc143's existing `vision_model`/`hybrid` Human Realism review remains the
only certifying pixel gate. It may return normalized evidence to the existing
bounded remote re-sign/retry flow. Local MCP remains an exact,
conversation-only comparison relay; it can demonstrate whether the signed
direction becomes more plausible, but cannot certify delivery.

## Acceptance

1. The re-sign payload contains the generic material-completeness instruction.
2. It reaches the actual `provider_prompt_human_naturalness_resign` call with
   the frozen shared Human Realism contract.
3. No local creative field, expression/material catalogue, template branch,
   or Provider-specific path is introduced.
4. A new exact Local MCP comparison may use only the fresh Brain-signed prompt
   and the same admitted reference bindings as Web materialization.
