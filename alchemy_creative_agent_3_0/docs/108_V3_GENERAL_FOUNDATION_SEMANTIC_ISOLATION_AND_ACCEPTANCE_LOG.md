## Status

This log records the General Template foundation corrective milestone completed
on 2026-07-12. It is an acceptance record for the defects discovered during
the Doc103 real-provider quality run. It is **not** a production declaration
for specialized templates or text-pixel delivery.

The governing release gates remain Doc104 Gate A--D and Doc107's production
requirements.

## Observed evidence and root causes

The original successful text-only quality samples made three General-layer
defects reproducible:

1. The short Latin industry keyword `spa` was matched as a raw substring of
   `atmospheric perspective`, selecting a Local Service vertical pack for a
   landscape.
2. `fair-faced concrete` was interpreted as a human `face` request. The
   General template therefore built a character suite recipe and strict
   portrait review despite no person reference or person capability in the
   frozen activation plan.
3. Several shared default paths used portrait language even when the subject
   was generic: selection-candidate guidance, strong-reference continuation,
   fallback-brain variation guidance, complex-prompt constraints, and strict
   review summaries. This was semantic leakage into provider/review/retry
   paths, not a provider-side behavior.

The real-image lower-right mark heuristic also classified dense, clean natural
and architectural detail as a possible mark. It was a local review false
positive; the reviewed images were clean.

## Implemented closure

- Latin intent and industry terms now require alphanumeric word boundaries;
  CJK matching remains literal.
- Character selection requires positive character evidence. A generic request
  now keeps a generic subject type unless a product or character evidence path
  is actually active.
- General suite direction, strict visual review, mode-quality profile, and
  strong-reference continuation now have explicit generic/product/character
  behavior. Portrait-only prompt, negative, review, retry, and user-visible
  terms are emitted only for the applicable character path.
- Fallback-brain candidate language and complex-prompt fidelity constraints
  are subject-neutral. Character-specific quality is still supplied by the
  activated Human Realism and portrait identity capabilities.
- The lower-right mark detector now requires a compact edge pattern, avoiding
  a veto on broad architectural geometry, rock texture, fabric texture, or
  other dense but unmarked detail while preserving the deliberate mark fixture.

These changes preserve the core governance rule:

```text
Frozen capability plan
-> active capability contribution only
-> provider prompt, review, retry, and visible summary use the same scope
```

## Remaining delivery gates

This milestone closes the then-observed text-only and controlled no-reference
General semantic-isolation and false-review defects. A later real
reference-conditioned Project Mode run exposed additional inactive
human/portrait prompt leakage and continuation-resolution defects, governed by
Doc109. The following requirements are deliberately still open:

| Gate | Required evidence before production declaration |
| --- | --- |
| Doc104 Gate C | Core owned-reference cases now pass and are recorded in Doc104: product truth, lifestyle, product-on-person, bounded review/retry, append-only history, and provider-failure recovery. The current fixture is unbranded, so visible label/logo fidelity remains a conditional future-fixture check. |
| Doc104 Gate D | The manual browser sequence has now run and failed: selected generated output materialization, duplicate provider input, truthful finalization, media-only result display, live pixel review, and neutral General UX must be corrected and rerun under Doc109. |
| Doc107 | An approved redistributable font manifest, required Tesseract locale data, and real OCR/layout evidence before text-pixel delivery is enabled. |

Until those gates have recorded evidence, E-Commerce text suites and other
specialized templates may develop against their contracts but must not be
marked production-ready. General remains scenario-neutral and must not expose
slot, platform, export-package, or named-specialist semantics.
