# Doc169 — Developmental-Transition Identity Evidence Contour Decoupling

Status: implemented; controlled Local MCP and shared Vision acceptance passed.

Related authority: Doc93–96, Doc103, Doc163, Doc167 and Doc168.

## 1. New evidence after Doc168

Doc168 correctly removed the complete source frame from a same-person request
whose developmental stage, complexion, camera, scene and finish belonged to the
current request. Fresh neutral and lively Local MCP outputs consequently lost
the old garden color/scene contamination and improved complexion, attention and
expression.

Both outputs nevertheless retained a longer and more tapered lower face than
the external five-to-six-year benchmark. The neutral output read around seven
to eight; the lively output around six to seven. The cause is visible in the
sole admitted `portrait_identity_crop`: although its outer scene is softened,
the image still exposes the complete source cheek, jaw and lower-face contour.
Image 2 can therefore obey the textual age decision while treating source-stage
soft-tissue geometry as hard visual identity evidence.

This is not a request for a round-child-face recipe. It is an evidence-authority
conflict during any explicit same-person developmental transition.

## 2. Governing rule

When the Remote Brain freezes:

```text
developmental_age_intent = current_request_assigns_stage
source_age_inheritance = not_automatic_when_current_prompt_assigns_age
```

an uploaded identity root must preserve stable internal feature relationships
without presenting stage-dependent outer face contour, soft-tissue volume,
source complexion or source capture context as hard Provider evidence.

Same-stage identity continuity remains unchanged and continues to use the Doc95
feature-detail plus head-geometry pair. Professional anchor preparation and its
2/3/5 reference budget remain unchanged.

## 3. Provider-only derivative

The shared reference service gains one generic provider-only derivative:

```text
portrait_identity_stage_flexible_feature_crop
```

It is selected only by the existing typed developmental-age decision, never by
age words, demographic classification, template, child/kidswear detection or
scene keywords.

The derivative:

- retains the localized internal identity-feature region needed to recognize
  the same person;
- strongly attenuates the outer facial silhouette, jaw/cheek contour, hair,
  source complexion and source-frame context;
- remains linked to the original asset hash and `portrait_identity_truth`;
- is ephemeral Provider evidence and is not a new user asset, candidate,
  delivery, biometric record or persistent face model;
- emits no prompt text and cannot alter Brain ownership.

The implementation may use the existing face-localized crop and feathered
isolation machinery. It must not add face landmarks, age estimation, ratio
thresholds, child-feature measurements or demographic templates.

## 4. Acceptance

Code acceptance requires:

1. cross-stage Standard/General identity work admits exactly the new
   stage-flexible feature derivative and suppresses the complete source frame;
2. same-stage Doc95 still admits feature-detail plus head geometry;
3. Professional root/supplementary evidence budgets are unchanged;
4. product and non-human identity paths are unchanged;
5. canonical Web/Local MCP reference hashes remain identical;
6. no local prompt authoring, age parser or child-specific branch exists.

Visual acceptance repeats both neutral and lively approximately-six-year
captures. The external boards remain evaluation-only and never enter Provider.
The target is not one fixed face shape: it is source identity that no longer
imports a conflicting source developmental contour. Human comparison remains
decisive for complexion, skin material, soft-tissue fullness, gaze, affect and
lower-face coherence.

## 5. Benchmark iteration evidence

The first implementation removed the complete source frame and produced one
stage-flexible feature derivative. A lively capture improved to approximately
the benchmark range, while two independent neutral captures still varied from
older-looking to near-target. Inspection of the derivative showed that a broad
feather around the internal-feature mask still exposed the source lower-face
silhouette as low-contrast pixels.

The second calibration therefore changes only evidence isolation: it more
strongly neutralizes and blurs the region outside the internal identity-feature
area and shortens the mask's lower reach. It does not add a developmental-stage
prompt, facial part target, age estimator or template branch. This calibration
is accepted only if the same Brain-authored neutral and lively directions both
retain identity while no longer reproducing the source-stage outer contour.

## 6. Controlled acceptance result

The external model boards were used only for human comparison and were never
sent to the Provider, copied into runtime metadata or registered as identity
assets. The only image evidence remained the authorized source asset and its
ephemeral stage-flexible derivative.

The controlled Local MCP cycle produced fresh neutral and lively candidates
from exact Remote-Brain-signed prompts and the single admitted derivative. The
first neutral samples and one lively sample exposed stochastic developmental
or expression drift. The existing shared review contract then supplied only
normalized observations and generic issue dimensions to one bounded Remote
Brain rewrite; no local repair sentence or facial recipe was appended.

The selected bounded-retry results achieved:

- neutral: `hybrid / verified / pass`, overall `0.92`, Human Realism `0.90`;
- lively: `hybrid / verified / pass`, overall `0.91`, Human Realism `0.93`;
- no shared Vision issue codes;
- strict external human comparison around `9.0/10`, in the same controlled
  quality band as the supplied five-to-six-year model references across
  complexion, skin material, soft-tissue fullness, gaze, affect and lower-face
  developmental coherence.

The result demonstrates the intended architecture: evidence authority prevents
source-stage pixel conflict; the Remote Brain remains the sole complete prompt
author; shared pixel review and one bounded rewrite absorb Image 2 sampling
variance. It does not establish a universal demographic face template or
permit benchmark images to become Provider evidence.
