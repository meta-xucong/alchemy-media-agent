# Doc150 — V3 Shared Human Presence Materiality and Expression Resolution

## Status

Implemented shared Human Realism refinement. It strengthens the existing
Doc147 semantic signing duty; Doc152 further clarifies that this duty allows
situation-owned smiles while rejecting a default camera-presentational smile.
Neither document adds a child, apparel, template, provider, or Local MCP path.

## Controlled observation

The fresh Doc149 blue-dress plan correctly activated Human Realism v5, reached
remote planning, canonical finalization and independent Human Naturalness
re-signing, and was relayed to Local MCP without any local prompt edit.

The resulting pixels nevertheless showed that the signed direction was still
too generic: it contained only broad "natural/candid" photography language
and a static standing pose. The image model then selected a familiar polished
display smile and clean beauty surface. This is not evidence that a young
person needs a special rule. The same under-specified personhood can affect
any real visible person.

## Decision

For an active shared `expression_ownership_requirement`, a complete Brain
direction must resolve the person's ordinary attention, awareness, or response
as part of the one scene it authors when the user did not explicitly supply an
expression. A generic natural/candid/photorealistic label plus a static pose
does not constitute an expression decision and cannot be approved at the
Human Naturalness re-sign stage.

The Brain remains free to choose the actual ordinary state from the complete
user-owned situation. It may preserve an explicit user expression and it may
retain a genuinely situation-owned smile. The change does not prescribe a
neutral face, a replacement expression, a face detail recipe, an ethnicity,
an age, or a palette.

## Hard boundaries

1. The Remote Brain is still the only author of the renderer prompt. Local
   runtime validates the frozen receipt and cardinality only.
2. No local keyword/regex detector may search the prompt for a smile, mouth,
   eye, pose, or demographic term; no local prompt suffix, negative prompt,
   expression catalogue, or face repair is permitted.
3. The existing shared `human_expression_context` vision/hybrid review remains
   the only pixel-level feedback channel. It can request the existing bounded
   re-sign/retry path but cannot create a special expression retry.
4. Doc148 complexion and photographic-material obligations remain unchanged.
   A person must be present as a coherent whole; this document does not turn
   skin, expression, or reference truth into independently stacked phrases.
5. Local MCP remains conversation-only and non-certified. Its result can
   expose a semantic-quality gap but cannot become a production delivery or
   replace a vision/hybrid certification.

## Acceptance

- The re-sign payload explicitly tells the remote Brain that a static generic
  description is insufficient, while retaining the no-catalogue rule.
- Source/contract regression proves the instruction reaches the actual
  `provider_prompt_human_naturalness_resign` request, not only planning prose.
- General, E-Commerce and Photography share the same Human Realism semantics;
  none acquires a vertical-specific expression path.
- A controlled comparison may use Local MCP only when it relays the fresh
  Brain-signed prompt and admitted reference binding without a rewrite.
