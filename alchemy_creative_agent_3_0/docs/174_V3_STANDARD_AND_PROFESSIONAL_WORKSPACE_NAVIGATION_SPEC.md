# Doc174 — V3 基础版与专业版工作台导航及视觉资产整合规范

状态：`authoritative_frontend_reconstruction`

基线：`main@237573a`

## 0. 结论与权威关系

V3 的“专业版”不是在基础项目创建页中增加一项开关，也不是新的 General
模板。它是一个独立工作台：先管理可复用、可审核、可版本化的视觉资产，再在
专业项目中显式选择资产。基础版保持现有的新建项目、模板、参考图、项目续作和
结果工作流，不显示或要求视觉资产。

```text
生图 V3.0 · Creative OS（标题栏入口）
  ├─ 基础版
  │   └─ 原有 V3 项目工作台，不改变普通用户路径
  └─ 专业版
      ├─ 视觉资产库：建立 / 准备 / 审核 / 启用 / 归档资产
      └─ 专业项目：选择模板、创建项目、选择一个或多个已启用资产
```

本文件替换 Doc173 的前端导航和页面层级描述；Doc173 的 Library ownership、
项目绑定、冻结、Reference admission、Remote Brain、Provider、Review、Retry
与 append-only history 合同继续有效。

## 1. 不可变边界

1. 基础版页面、模板语义和普通项目创建流程保持不变；不因关键词、上传图或
   已有资产自动进入专业版。
2. 专业版不是第二套 Provider、Brain、Review、Retry、存储或图片库。它只管理
   资产生命周期和项目级绑定。
3. 新 Job 的最终 provider prompt 始终由 Remote Brain 基于受保护用户意图、模板
   合同、冻结资产证据、CapabilityActivationPlan 与 Constraint Ledger 完整输出。
   前端和资产库不得拼装 prompt、负面词、年龄/皮肤/镜头/场景配方。
4. 当前一期仅可创建和选择 People Asset / Face Identity。产品、场景、品牌资产
   的卡片、按钮和伪入口都不提前展示；未来只扩展同一 BindingSet 和通道所有权。
5. 项目可以绑定零个或多个不冲突资产。首期只能有一个 People Asset，因为其
   身份通道相互冲突。未来“人物 + 产品”是两个不同资产类型的显式绑定，不是
   合成参考图或本地结构化规则。

## 2. 标题栏入口

原“生图 V3.0 · Creative OS”改为可访问的菜单触发器。点击或键盘操作后弹出：

```text
基础版
  原有项目工作台：通用、电商、摄影师模板和最近项目

专业版
  建立视觉资产，并在专业项目中把已启用资产固定到后续任务
```

选择项要有明确焦点、Esc 关闭、点击外部关闭、`aria-expanded` 状态以及可刷新的
路由状态。`/creative-agent-v3` 默认进入基础版；`?workspace=professional` 直接
进入专业版。任何目录或 API 失败均不得把用户切换到另一个工作台。

## 3. 基础版

基础版必须保持用户既有心智模型：

```text
选择 General / E-Commerce / Photography 模板
→ 写自然语言目标
→ 创建项目
→ 上传普通参考、生成、选择结果、继续制作
```

基础版不显示“视觉资产库”按钮、资产建立表单或“本项目视觉资产”面板。它不会
自动使用资产库中的人物、产品或场景；普通上传参考仍按现有模板与 Doc93 处理。

## 4. 专业版

专业版是一张独立首页，按从上到下的顺序呈现：

### 4.1 视觉资产库

上半部分是资产库。用户可以：

```text
新建人物资产
→ 上传已经授权的源图
→ 填写自然语言建模意图
→ 开始标准建模
→ 查看正面 / 侧前方 / 侧面状态
→ 明确确认启用版本
```

资产卡只显示名称、类型、易懂状态、最近使用和用户可执行的下一步；不显示
prompt、路径、hash、候选、供应商、内部审查分数或 Job ID。上传源图不等于启用。

### 4.2 专业项目

资产库下方使用与基础版相同、但明确标注为“专业项目”的模板选择、目标输入、
创建按钮和最近项目卡。创建项目时写入的只是用户显式选择的
`v3_workspace=professional` 表面归属，不写旧 `professional_mode`、`people_asset_id`
或任何资产证据。

项目创建后，项目概览顶端固定显示：

```text
本项目的视觉资产
[选择视觉资产]
```

选择弹层只列出当前用户有权使用、当前模板可接受、已启用且证据完整的资产。用户
可确认新增、替换或移除绑定；每次改动只影响未来 Job。无资产绑定是允许的项目
状态，但不应被误称为“专业能力已启用”。若用户明确绑定的资产过期、冲突或失效，
新的 Job 必须在 Provider 前 blocked，并给出“选择其他资产 / 解除该绑定”的下一步，
绝不静默切回基础版或普通参考。

## 5. 运行时与历史兼容

```text
专业项目当前 BindingSet
→ 服务端校验 owner、版本、同意来源、模板兼容性和通道冲突
→ FrozenVisualAssetBindingSet
→ Reference admission / Brain / Envelope / Ledger
→ Shared Provider / Review / bounded retry / final delivery
```

项目的工作台归属不是生成模板，也不是创作方向。General、E-Commerce 和 Photography
仍各自拥有模板语义；资产库只提供资产事实和通道所有权。

历史项目保持可读。历史 `professional_mode` 与 project-scoped People Asset 写入只做
读取兼容，不得进入新的 Brain、Envelope、Provider、普通 UI 或前向写入。旧项目若
已有可验证的 Library Binding，可以在专业版以历史项目方式打开；不可验证记录保持
只读，不自动提升或迁移。

## 6. 验收矩阵

1. 标题栏点击后可选择基础版或专业版，刷新、直达链接、Esc 和窄屏都正确。
2. 基础版 DOM 和用户路径不出现资产库/绑定控件；原模板卡和项目列表不回归。
3. 专业版先呈现资产库，随后呈现专业项目；人物资产完整生命周期只从专业版进入。
4. 新建专业项目不写旧 `professional_mode` / `people_asset_id`；其项目元数据只记录
   `v3_workspace=professional`。
5. 标准和专业项目列表相互隔离；打开项目时按其记录恢复正确工作台。
6. 只有专业项目显示资产绑定面板；确认绑定、解除、刷新和重新打开均与服务器状态
   一致；历史 Job 不变。
7. General、E-Commerce、Photography 在专业项目中保持各自模板语义；不新增任何
   私有生成、审查、重试或结果选择路径。
8. 1440 / 1024 / 430 / 390 四种尺寸下，标题菜单、资产生命周期、项目创建、绑定
   弹层、刷新和错误状态均可完成，且无 HTML/内部错误泄漏。
