# Doc161 — V3 Reference Ownership Brain Sign-off and Identity Evidence Isolation

Status: implemented. The bounded Doc162 M5 rerun proved that the v2 exterior-context
isolation profile was still too broad for a full-body identity source: source wardrobe
and hairstyle pixels remained visible inside the retained center. Doc163 is the current
authority for face-localized identity evidence; this document remains authoritative for
Brain ownership sign-off and the prohibition on prompt-word stacking.

## 1. Problem statement

The Doc160 rerun proved that normalized pixel observations reached the Remote Brain and produced a distinct rewritten canonical prompt. It still produced no acceptable Professional front winner.

The sanitized runtime record and local artifact inspection exposed three independent facts:

1. The frozen Doc93 reference policy correctly assigned identity geometry to the portrait reference and assigned hair, wardrobe, light, scene, camera, mood and finish to the current request.
2. The final Brain prompt nevertheless described source hair as preserved. The finalizer received only opaque reference IDs and roles, not the resolved channel-ownership package, so it could not formally sign that boundary.
3. Provider-only identity crops still retained a recognizable source hairstyle and garment outside the useful facial region. Textual ownership alone therefore competed with visually stronger reference evidence.

The same run was also not a formal M5 proof: its temporary host did not carry the existing Professional Face Identity quality contract into the canonical finalizer. It validated the generic shared path and Doc160, but not the complete Professional contract.

## 2. Authority and design boundary

This document extends, and does not replace:

- Doc93 for reference-channel ownership;
- Doc95/96 for complementary portrait identity evidence and high-fidelity execution;
- Doc113 for frozen runtime truth;
- Doc134/135 for Brain-owned final Provider language;
- Doc160 for review-evidence-to-Brain repair.

The correction is foundation work. It must remain useful for adult portraits, child portraits, Photography, Professional Mode and ordinary same-person General jobs. It must not create a child, kidswear, hairstyle, wardrobe or scene-specific branch.

## 3. Required implementation

### 3.1 Brain-owned reference ownership decision

When a frozen `resolved_reference_policy_package` applies, ScenarioRuntime projects only these typed facts into finalization:

- reference-owned channels;
- current-request-owned channels;
- explicit user locks;
- blocked reference inheritance channels;
- the frozen envelope/ledger binding.

Legacy `provider_prompt_rules`, negative rules, retry phrases and local prompt fragments never cross this boundary.

The Remote Brain must reconsider the complete canonical prompt and return a schema-only receipt:

```text
contract_version = v3_reference_channel_ownership_decision_v1
status = approved | rewritten
owner = remote_v3_llm_brain
```

Missing or malformed receipts block before Provider materialization. The runtime must not inspect the prompt with keyword or regex rules and must not append a repair sentence.

### 3.2 Provider identity evidence isolation v2

For identity-only portrait references whose hair, wardrobe, lighting, scene or style channels remain prompt-owned:

- retain facial feature and head-geometry evidence in complementary derivatives;
- strongly desaturate, lower contrast, blur and neutralize the exterior context;
- tighten the retained face region so garment and hairstyle pixels no longer act as hidden owners;
- keep the user upload unchanged;
- keep explicit reference-owned hair or appearance locks on the existing preservation path.

This is evidence admission, not creative direction. It does not invent appearance, add renderer wording or alter product/reference truth.

### 3.3 Formal Professional anchor-preparation context

Anchor preparation must use a formal typed planning context even before an active pack exists. It carries:

- Professional Mode selection;
- preparation view role;
- shared serial-reference strategy;
- existing Face Identity quality contract;
- Remote Brain and shared Vision ownership.

It carries no raw reference path, complete prompt, candidate pixels, private binding or caller-authored pack. A Professional real-image request missing the typed quality contract fails closed before final prompt signing.

## 4. Explicit non-goals

This change must not:

- add child/kidswear/fashion keyword detection or a scene recipe;
- add a local prompt builder, negative-word stack, regex prompt validator or retry phrase;
- weaken shared Vision thresholds or reinterpret `fail_retryable` as pass;
- add a Provider, reviewer, retry loop, storage path or Professional-private generation flow;
- change Standard Mode reference ownership;
- make a failed M5 run count as Gate C/D or production acceptance.

## 5. Acceptance matrix

Code acceptance requires:

1. Finalizer payload exposes the typed ownership decision and requires a receipt.
2. Missing receipt blocks before Provider.
3. Renderer-rule strings from the legacy Doc93 package do not enter Brain finalization.
4. Prompt-owned identity derivatives use isolation v2 and materially suppress exterior color/context while retaining facial evidence.
5. Explicit user locks on reference-owned hair/appearance preserve the assigned channel.
6. Product-only/nonhuman references are unchanged.
7. Professional anchor preparation carries the typed quality contract, and Professional execution without it fails closed.
8. General, E-Commerce and Photography isolation regressions remain green.
9. Full V3 regression, compile, browser syntax and diff checks pass.

Pixel acceptance is one new, independent, bounded Professional front run:

```text
3 front candidates
→ shared Vision
→ at most one bounded repair for the first retryable candidate
→ winner only if verified and deliverable
```

The run must record the Professional quality contract, Doc93 ownership receipt, prompt/reference parity, v2 derivative profile and append-only review history. If no front winner passes, three-quarter/profile and pack activation remain blocked; evidence is sealed and the gate is not relaxed.
