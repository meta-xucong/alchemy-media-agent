## 3. Visual baseline assessment

The accepted comparison shows a plausible East Asian child with correct age
presentation, credible limbs/hands, a coherent outdoor light field and a
recognizably preserved blue ruffled dress. It is accepted as a functional
Human Realism baseline, not as a final aesthetic ceiling.

The remaining quality gap is general rather than garment- or child-specific:
the image is still slightly too polished and catalogue-like; the face,
expression, hair and background retain some idealized synthetic regularity;
the printed graphic and layered skirt are plausible rather than SKU-exact.
The next refinement must therefore improve the shared remote-Brain/review
quality judgment across several materially different real-person scenes. It
must not add subject keywords, prompt checklists, child/kidswear branches,
template recipes or local renderer prompt edits.

## 4. Explicitly not accepted by this phase

- Web Provider/Gateway stability, upstream timeout behavior and Provider Gate
  certification.
- Vision-model or hybrid final-pixel certification of this comparison.
- General browser Gate D, Photography P10 or E-Commerce Gate C/D production
  approval.
- SKU-grade textile/print matching or multi-image diversity claims.

The desktop MCP transport in the current Codex process was intentionally
restarted during diagnosis and did not auto-reconnect. The launcher/facade was
verified with the same configured environment and source tree; a Codex Desktop
restart is an operational reconnect only, not a code or acceptance failure.

## 5. Follow-up acceptance rule

Before asserting an aesthetic improvement, test at least three distinct
real-person contexts and compare the frozen remote prompt, active shared
capabilities, real review verdict and final selected result. A single blue
dress remains a regression fixture, never the basis for a new shared branch.

## 6. Successor authority

quality phase: a more accountable existing Brain re-signing receipt, shared
pixel-feedback closure, and retirement of executable local retry phrasing.
This document remains the accepted planning/parity baseline; it does not claim
that the later quality phase is already implemented or pixel-certified.
