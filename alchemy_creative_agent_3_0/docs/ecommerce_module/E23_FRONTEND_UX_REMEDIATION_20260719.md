# E-Commerce 前端 UX 修复合同（E23）

状态：`implementation_required`

基线：`origin/main@56fbac6`

共享状态与新手交互遵循 Doc172。本文件只处理 E-Commerce 的事实输入、数量、结果和
失败恢复，不改变远端 Brain、GPT Image 2、共享 Review/Retry 或生产门禁。

## 1. 必须清除的旧语义

普通电商界面不得再承诺固定的：

```text
主图、卖点图、场景图、细节图、信任图
```

也不得把旧 `preset`、`slot`、`recipe`、`overlay` 作为新任务的可见选择或结果标题。
历史字段可以只读兼容，但不能进入新的 Brain 请求、冻结计划、Provider prompt 或普通
交付面。

电商页面应解释为：

```text
你提供商品事实、明确需求和平台信息；系统会为每张图生成一个完整的画面方向。
```

## 2. 新手主路径

```text
选择“电商创意”
→ 上传商品参考图
→ 写一句想要的画面
→ 可选补充平台、市场和商品事实
→ 选择生成数量
→ 提交
→ 查看每张完整图片
→ 对失败/需要确认的图片获得明确下一步
```

默认只显示必要字段：商品参考图、需求、数量。平台、市场、事实和谨慎宣传说法放在
“补充信息”中，并明确哪些是必须、建议填写和可选。

## 3. 数量合同

界面必须支持当前合同的 `1 / 2 / 4 / 7`，但不能自行假定所有平台都支持 7 张。

- 数量选项来自共享能力/平台合同投射；
- 用户选择数量后显示“本次准备生成 N 张”；
- 能力不支持时在提交前明确阻断或让用户选择可用数量；
- 不得静默把 7 张截成 4 张；
- 结果区显示实际最终交付数量和未交付数量。

## 4. 表单语义

当前 `v3BrandToneInput` 的值被提交为 `core_selling_points`，必须修正：

- 店铺视觉感受只进入用户视觉偏好/visual tone；
- 商品卖点必须有独立、明确的事实输入；
- 不得让用户以为“清爽、克制、明亮”是商品功能或销售承诺；
- 关键词、竞品、平台证据不能在前端拼成静态创意 recipe。

## 5. Brain 与结果状态

Brain 不可用、计划不完整、计划数量不一致或参考图不可用时，界面应显示：

```text
暂时无法整理这次电商需求，项目已保存。
请检查商品图或补充信息后重新尝试。
```

禁止显示“套图准备中”后永久悬挂，也禁止生成本地静态降级套图。

结果区只显示当前最终整图方向的最终交付：

- 每张图可预览、下载和继续使用；
- held/manual confirmation 显示为“需要确认”，不算完成；
- blocked/failed 显示人话原因和下一步；
- retry 历史折叠，不出现在普通数量统计；
- 不显示 slot、recipe、candidate、asset ID 或内部 provenance。

`renderV3ProjectNextActions` 必须恢复并为 blocked、held、completed 提供相应动作：

- 补商品事实；
- 更换参考图；
- 重新尝试一次；
- 返回项目；
- 下载当前已完成图片。

## 6. 响应式与防误操作

- 生成按钮点击后立即禁用并显示“正在准备”；
- 刷新/重新打开后从服务端恢复项目状态；
- 390/430 宽度下上传、数量、提交、结果预览和刷新按钮均可见；
- 不依赖 hover；
- 长平台名称和失败说明可换行；
- 结果图片加载失败时显示“重新加载图片”和“稍后刷新项目”。

## 7. 必须补的回归

- 新电商任务不出现旧固定套图文案；
- General 页面不出现电商平台、数量和套图字段；
- `1/2/4/7` 数量合同在浏览器和 API projection 中一致；
- tone 不再进入 selling points；
- Brain fail-closed、数量不一致、参考图阻断均有可理解恢复路径；
- 历史 recipe/slot/overlay 只读兼容，不回放到新任务和普通结果；
- 刷新、重开、重复点击不会改变任务数量或模板；
- 1440/1024/390/430 浏览器路径通过；
- 不新增电商私有 Provider、Review、Retry 或文字合成链路。

## 8. 完成标准

电商前端只有在新手能从商品参考图和一句话需求完成一次明确、可恢复的任务，并且
看不到固定套图承诺、旧 slot/recipe 语义和内部错误时，才可标记
`ecommerce_frontend_ux_passed`。这不等于 Gate C/D 或生产可用。
