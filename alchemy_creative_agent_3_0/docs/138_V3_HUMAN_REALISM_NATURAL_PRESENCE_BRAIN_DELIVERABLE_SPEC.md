# Doc138 - V3 Human Realism Natural-Presence Brain Deliverable

Status: **active foundation background; semantic v2 payload superseded for fresh enforced jobs by Doc143.**

This document extends Docs 91-96, 124, 128, 134-137. It records the
controlled blue-dress Local MCP comparison after Doc137 and closes the gap
between a valid Brain preflight receipt and a sufficiently substantive
whole-image human direction. It is not a child, apparel, E-Commerce,
Photography, or General Template rule.

## 1. Controlled finding

The current blue-dress result proves that the correct common path is active:

```text
admitted product truth
-> enforced visible-person evidence
-> shared Human Realism
-> remote Brain plan and final canonical prompt
-> exact Local MCP prompt/reference relay
-> one Codex ImageGen conversation image
```

The result is materially more credible than the earlier comparison: age,
hands, garment construction, light and scene integration are coherent. It is
not yet the quality target. The face still trends toward an overly even,
generic child-fashion archetype, with skin and expression more polished than
an ordinary real-camera moment.

> **Current implementation note:** Doc139 turns this finding into a second,
> independent remote-Brain re-signing pass for every active shared Human
> Realism contract. It supersedes any interpretation that a Doc138 receipt by
> itself proves the final canonical direction was substantively reconsidered.
> **Current implementation note:** Doc142 adds an additive, schema-only
> `approved|rewritten` decision receipt for that same re-signing pass. It does
> not create a third Brain call.

That observation does **not** justify a list of face, skin, hand, child or
camera fragments. The failure is semantic: a finalizer can return an
`approved` receipt while treating the request as a generic commercial beauty
image instead of deciding how a particular person should exist naturally in
the requested photographic situation.

## 2. Ownership and scope

```text
frozen evidence + shared semantic contract
-> remote Brain judges natural human presence in the whole composition
-> remote Brain writes the entire final renderer prompt
-> Provider or Local MCP relays that exact prompt unchanged
```

The Human Realism layer may contribute only a compact, reusable semantic
priority. It may not decide a face, pose, crop, smile, lighting recipe, age
style, fashion treatment, or product composition. It must not append a
renderer phrase after the Brain signs.

This rule applies equally to adults, children, people wearing products,
portraits and ordinary lifestyle scenes. An explicit young-person signal is
still only the existing safety boundary; it does not create a visual route.

## 3. Historical Human Realism semantic contract v2

For a real visible person, the frozen Human Realism contract contains two
additional non-creative decisions:

```json
{
  "natural_presence_priority": "individual_human_presence",
  "aesthetic_boundary": "preserve_user_style_without_generic_beauty_substitution"
}
```

They are not prompt text. They mean that the Brain must preserve the user's
requested mood and legitimate attractiveness while avoiding a default,
interchangeable beauty or child-fashion archetype when it writes the complete
canonical direction.

The contract stays deliberately small. It does not encode facial features,
skin locations, body parts, nationalities, age groups, product categories or
camera recipes.

## 4. Required Brain deliberation

When Doc137 preflight is required, the remote Brain must make one holistic
decision before it signs each output:

- Whether the direction represents a specific human being physically present
  in the requested scene, rather than a polished generic beauty archetype.
- Whether a candid, ordinary or lifestyle request retains an unforced
  expression, plausible physical presence and lighting/material continuity.
- Whether an explicitly glamorous or editorial request still keeps human
  particularity instead of introducing synthetic beautification.

The Brain incorporates the resolution into its one complete final prompt in
its own language. It returns the existing Doc137 audit receipt and the
same-pass Doc142 Human Naturalness decision receipt. It must not
emit a checklist, inner reasoning, issue code, negative prompt or a separate
repair phrase. Merely echoing generic adjectives such as "natural", "candid"
or "photorealistic" is not a completed semantic resolution.

## 5. Pixel-review and retry boundary

Text-only preflight is a forward creative safeguard, not proof that pixels
are good. The normal Web path remains:

```text
generated pixels
-> shared vision_model/hybrid Human Realism dimensions
-> normalized evidence only
-> one bounded Brain re-signing
-> append-only winner selection
```

Local MCP intentionally stops before review/retry/delivery. It can verify the
same canonical prompt/reference materialization and make a controlled visual
comparison; it cannot certify a production result.

## 6. Blue-dress assessment guide

Use the blue dress only as a cross-scene regression sample, alongside adult
portrait, adult product-on-person and non-person product cases. Score the
result holistically, not by counting prompt terms:

| Dimension | Current controlled result | Target |
| --- | --- | --- |
| Natural facial presence | improved but still polished/generic | recognizably real, particular and age-appropriate |
| Physical/scene coherence | good | consistently credible under varied light and pose |
| Garment truth | good upper garment; framing remains prompt-owned | preserve supplied truth without shared crop rules |
| Overall commercial realism | roughly 7.8/10 | Lovart-quality comparison band, about 8.5/10 or better |

The target is an expert visual comparison, not a claim of pixel certification
or a deterministic numerical metric.

## 7. Prohibited remedies

- local positive/negative word stacks for pores, hands, smiles, faces or
  children;
- child/kidswear branches, Providers, review routes or retry paths;
- regex/keyword outputs deciding whole-image naturalness;
- static face, pose, crop, lighting or garment recipes;
- treating Local MCP images as a production review or delivery;
- weakening Doc137's fail-closed Brain sign-off.
